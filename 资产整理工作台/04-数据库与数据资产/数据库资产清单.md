# 数据库与数据资产清单

- 建立日期：2026-04-28
- 作用：区分正式 schema、生成 seed、本地数据和数据治理脚本。

## 正式数据库资产

| 资产 | 路径 | 身份 | 后续动作 |
|---|---|---|---|
| D1 migrations | `infra/cloudflare/d1/migrations` | 正式 schema 资产 | 严格版本化 |
| D1 source seeds | `infra/cloudflare/d1/seeds/source` | 可读源种子 | 保持 README 和生成规则 |
| D1 generated seeds | `infra/cloudflare/d1/seeds/generated` | 生成物 | 明确是否全部入版本 |
| D1 rebuild SQL | `infra/cloudflare/d1/rebuild` | 治理/修复候选 | 标注一次性或长期 |

## 本地与参考数据

| 资产 | 路径 | 身份 | 后续动作 |
|---|---|---|---|
| 本地 SQLite | `var/local/info_collector.db` | 本地参考 / 过渡数据 | 不迁移到正式资产 |
| demo mock data | `apps/web/demo/mock-data` | demo / bootstrap 种子 | 与正式 D1 分开管理 |
| Python virtual data | `app/services/data/virtual_data.py` | 兼容/退役过渡层 | 不作为正式成功回退 |

## 数据资产规则

1. migration 是正式 schema，不与 seed 混淆。
2. generated seed 原则上不手工修改。
3. 本地 SQLite 不作为正式云端事实源。
4. demo 数据只能服务 demo / bootstrap / 示例，不作为正式业务真源。
