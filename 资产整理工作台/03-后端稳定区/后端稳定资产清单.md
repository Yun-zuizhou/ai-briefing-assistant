# 后端稳定资产清单

- 建立日期：2026-04-28
- 作用：明确正式在线后端、参考后端和测试资产边界。

## 正式在线后端

| 资产 | 路径 | 当前身份 | 后续动作 |
|---|---|---|---|
| Workers 应用 | `apps/edge-worker` | 正式在线后端 | 保持稳定 |
| 路由层 | `apps/edge-worker/src/routes` | 正式 API 面 | 新增正式 API 默认落这里 |
| 服务层 | `apps/edge-worker/src/services` | 正式业务逻辑 | 保持契约驱动 |
| 工具层 | `apps/edge-worker/src/utils` | 鉴权、DB、错误处理等 | 保持小而稳定 |
| Workers 测试 | `apps/edge-worker/tests` | 后端回归资产 | 继续补权限和深度场景 |

## 参考后端

| 资产 | 路径 | 当前身份 | 后续动作 |
|---|---|---|---|
| Python app | `app` | 离线 / 参考 / 验证链 | 不再作为正式在线主链 |
| Python API | `app/api/v1` | 参考接口 | 修改时需明确不是正式扩面 |
| Python services | `app/services` | 离线领域逻辑和兼容层 | 保留验证价值 |
| SQLAlchemy models | `app/models` | 本地参考模型 | 与 D1 正式 schema 区分 |
| Crawler | `app/crawler` | 离线采集逻辑 | 与 Workers 在线 API 区分 |

## 后端边界规则

1. 正式在线能力优先落 `apps/edge-worker`。
2. 共享前后端页面数据结构优先落 `packages/contracts`。
3. 数据结构变更必须对齐 `infra/cloudflare/d1/migrations`。
4. `app` 修改默认视为参考链或验证链修改。
5. 不把 Python 的虚拟数据回退重新作为正式成功路径。
