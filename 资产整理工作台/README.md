# 资产整理工作台

- 建立日期：2026-04-28
- 作用：在不破坏当前可运行仓库的前提下，建立资产迁移、整理、查找和缺口追踪的工作区。

## 使用原则

1. 本目录先承接清单和判断，不直接替代正式源码目录。
2. 任何真实文件迁移前，先在 `00-迁移清单/资产总表.md` 标记来源、目标分区、状态和后续动作。
3. 对旧文件、归档文件和 demo 文件，默认先登记为“待整理”或“参考候选”，不直接迁回正式主线。
4. 正式运行主线仍以仓库现有目录为准：
   - `apps/web`
   - `apps/edge-worker`
   - `packages/contracts`
   - `infra/cloudflare/d1`
   - `tools/scripts`
5. 运行产物、依赖目录和缓存目录不迁移，只记录忽略和清理策略。

## 目录说明

| 目录 | 用途 |
|---|---|
| `00-迁移清单/` | 总表、待整理项、缺口、决策记录 |
| `01-正式运行主线/` | 记录正式前端、正式后端、共享契约等主线资产 |
| `02-UI继续优化区/` | 记录仍需继续优化的页面、组件、样式、demo 候选 |
| `03-后端稳定区/` | 记录 Workers 后端稳定资产和边界 |
| `04-数据库与数据资产/` | 记录 D1 schema、seed、SQLite、本地数据边界 |
| `05-链接方式与运行规范/` | 记录启动、测试、部署、API 连接和 secret 规范 |
| `06-文档事实源/` | 记录当前文档事实源与瘦身目标 |
| `07-参考实现与候选资产/` | 记录 Python 参考链、归档区、demo 候选资产 |
| `99-隔离与待删除/` | 记录缓存、运行产物、旧结构残留、准备删除项 |

## 当前第一批动作

1. 先维护 `00-迁移清单/资产总表.md`。
2. 再从 `待整理清单.md` 按批次检查旧文件。
3. 每发现缺口，写入 `缺口清单.md`。
4. 每做一次是否迁移、是否保留、是否删除的判断，写入 `迁移决策记录.md`。
