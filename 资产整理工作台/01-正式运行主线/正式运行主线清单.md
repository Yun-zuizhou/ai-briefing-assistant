# 正式运行主线清单

- 建立日期：2026-04-28
- 作用：记录当前不能随意移动、不能误删、不能被 demo 或旧结构替代的正式运行资产。

## 主线资产

| 资产 | 路径 | 身份 | 管理规则 |
|---|---|---|---|
| 正式前端 | `apps/web` | 当前唯一正式前端 | 不整体搬动；UI 优化在原目录小步完成 |
| 正式在线后端 | `apps/edge-worker` | 当前正式 Workers 后端 | 新增正式 API 默认落这里 |
| 共享契约 | `packages/contracts` | 前后端共享页面契约单一源 | 变更需同步前端和 Workers |
| D1 数据库结构 | `infra/cloudflare/d1/migrations` | 正式 schema | 变更必须可追踪 |
| 统一脚本 | `tools/scripts` | 运行、测试、数据准备入口 | 不恢复旧根级 `scripts` |
| 根级命令 | `package.json` | 一键命令分发中心 | 保持 `task:*` 稳定 |
| 自动化测试 | `tests` / `e2e` / `apps/edge-worker/tests` | 回归和验收资产 | 主门槛仍以 `npm.cmd run check` 为准 |

## 不应替代主线的资产

| 资产 | 路径 | 原因 |
|---|---|---|
| Python app | `app` | 当前是离线 / 参考 / 验证链 |
| demo 页面 | `apps/web/src/demo` | 不属于正式前端页面主线 |
| demo mock data | `apps/web/demo/mock-data` | 不属于正式业务数据源 |
| 归档区 | `_archive_workspace` / `文档/归档` | 只用于追溯和候选吸收 |
| 过时文档 | `文档/过时` | 不作为当前执行依据 |

## 主线变更前检查

1. 是否会影响 `npm.cmd run check`？
2. 是否会改变前端到 Workers 的连接方式？
3. 是否会改变 D1 schema 或 seed 策略？
4. 是否需要同步 `packages/contracts`？
5. 是否需要同步 `文档/进行中/当前测试与验收总表.md`？
