# 运行入口与链接方式

- 建立日期：2026-04-28
- 作用：把当前项目的启动、验证、前后端连接、数据库连接和生产配置入口集中到一页。

## 当前正式链路

```text
apps/web(Vite + React)
  -> apps/edge-worker(Cloudflare Workers + Hono)
  -> Cloudflare D1
```

## 辅助参考链路

```text
app(FastAPI / Python reference)
  -> var/local/info_collector.db
```

该链路只作为离线、参考、验证和兼容层，不再作为正式在线主后端。

## 标准命令

| 场景 | 命令 | 说明 |
|---|---|---|
| 初始化 | `npm.cmd run task:init` | 等价入口来自根 `package.json` |
| 启动完整开发链 | `npm.cmd run task:start` | 默认走 Workers + Web |
| 快速启动 | `npm.cmd run task:start:fast` | 快速开发入口 |
| 只启动前端 | `npm.cmd run task:start:web` | 前端本地调试 |
| 只启动 Workers | `npm.cmd run task:start:workers` | 正式在线后端本地调试 |
| 启动 Python 参考链 | `npm.cmd run task:start:python:lite` | 离线/参考链 |
| 主门槛验证 | `npm.cmd run task:verify` | 当前主检查入口 |
| 门禁摘要 | `npm.cmd run task:verify:proof` | 更新测试文档前优先使用 |
| Workers 配置预检 | `npm.cmd run task:access:workers` | 检查本地 vars 和 secret 前提 |
| D1 准备 | `npm.cmd run task:data:prepare` | 生成 / 准备 D1 数据资产 |

## 当前主门槛

唯一主门槛仍是：

```powershell
npm.cmd run check
```

测试数字只维护在：

```text
文档/进行中/当前测试与验收总表.md
```

## 数据库连接边界

| 数据库 | 路径 / 服务 | 身份 |
|---|---|---|
| Cloudflare D1 | `infra/cloudflare/d1` | 正式数据库资产 |
| 本地 SQLite | `var/local/info_collector.db` | 过渡 / 本地参考数据 |
| D1 migrations | `infra/cloudflare/d1/migrations` | 正式 schema |
| D1 generated seeds | `infra/cloudflare/d1/seeds/generated` | 生成物，需要版本策略 |

## 配置与 secret

| 配置 | 位置 | 规则 |
|---|---|---|
| 前端本地 env | `apps/web/.env` | 本地生成，不作为生产 secret |
| Workers 本地 vars | `apps/edge-worker/.dev.vars` | 本地使用，不进仓库 |
| Workers vars 示例 | `apps/edge-worker/.dev.vars.example` | 可版本化 |
| 生产 secret | Wrangler secret | 不写入仓库 |
| 根 `.env` | `.env` | 本地配置，不进仓库 |

## 当前禁止回退

1. 不把正式在线 API 新增回 `app`。
2. 不恢复根级 `wrangler.toml` / `worker.js` 作为正式入口。
3. 不把 demo 路由重新挂回正式 `BrowserRouter`。
4. 不把 `apps/web/demo/mock-data` 当作正式业务真源。
5. 不手工维护可生成 seed，除非已明确记录原因。
