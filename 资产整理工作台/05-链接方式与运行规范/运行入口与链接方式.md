## 1. 一键命令速查

```bash
# 环境初始化（首次使用或环境变更后）
npm.cmd run task:init          # 等价 setup：安装依赖 + 补齐 .dev.vars

# 启动开发环境
npm.cmd run task:start         # 等价 dev：默认 Workers 后端 + 前端
npm.cmd run task:start:fast    # 跳过 scheduler 快速启动
npm.cmd run task:start:web     # 只启动前端
npm.cmd run task:start:workers # 只启动 Workers 后端
npm.cmd run task:start:python  # Python 参考后端 + 前端
npm.cmd run task:start:python:lite  # Python 轻量模式

# 验收门禁
npm.cmd run task:verify        # 等价 check：主门槛
npm.cmd run task:verify:proof  # 门禁留痕摘要
npm.cmd run task:verify:workers  # Workers 类型检查 + 测试
npm.cmd run task:verify:e2e    # E2E 浏览器测试

# 数据访问
npm.cmd run task:access:d1     # D1 凭证引导配置
npm.cmd run task:access:workers  # Workers secret 核验
npm.cmd run task:access:consult  # AI digest provider 同步

# 数据准备
npm.cmd run task:data:prepare  # 生成 D1 seeds
npm.cmd run task:data:sync-demo  # 同步 demo 数据
```

## 2. 端口与服务

| 服务 | 端口 | 启动方式 | 说明 |
|------|------|----------|------|
| 前端 Vite dev server | `5173` | `npm run dev` | 默认启动，HMR |
| Workers 后端 | `8787` | `npm run dev:backend:workers` | `wrangler dev`，默认后端 |
| Python 参考后端 | `8787` | `npm run dev:python` | uvicorn，bootstrap 兼容 |

前端通过 Vite proxy 将 `/api/v1` 和 `/api-config` 代理到 `http://127.0.0.1:8787`，开发时只访问 `localhost:5173`。

## 6. 环境变量要点

### 前端 (`apps/web/.env`)
| 变量 | 说明 |
|------|------|
| `BACKEND_PORT` | 后端端口，默认 `8787` |

### Workers (`apps/edge-worker/.dev.vars`)
| 变量 | 说明 |
|------|------|
| `INTERNAL_API_TOKEN` | system 写入端点鉴权，本地默认 `dev-internal-token` |
| `REQUEST_LOGGING` | 请求日志开关 |
| `ERROR_LOGGING` | 错误日志级别 |

### Workers 生产 secret
| Secret | 说明 |
|------|------|
| `INTERNAL_API_TOKEN` | 生产 internal token（`wrangler secret put`） |

### Python 参考链 (`.env`)
`AI_PROVIDER` 支持：`deepseek` / `openai` / `nvidia` / `anthropic` / `gemini` / `zhipu` / `qwen` / `local`

## 9. 日常工作流

```bash
# 启动开发
npm run task:start                 # Workers + 前端 → http://localhost:5173

# 改完验证
npm run task:verify                # 主门槛全绿

# 首次接入 D1
npm run task:access:d1             # 配置凭证
npm run db:d1:verify-mainline      # 验证连通

# 部署前
npm run task:verify                # 主门槛
npm run task:verify:workers        # Workers 门槛
npm run task:access:workers        # 确认 secret 齐备
```
