# 命令规范说明

- 更新日期：2026-04-24
- 作用：存放当前仓库可直接复制执行的一键命令入口，减少重新研究脚本与试错成本

## 1. 使用原则

1. Windows 下默认直接复制：
   - `npm.cmd run task:...`
2. 命令入口只做统一收口，不额外创造第二套实现逻辑
3. 若底层实现变动，优先保持 `task:*` 命名空间稳定
4. 需要更新“测试通过”类文档口径时，先执行 `npm.cmd run task:verify:proof`，再写结论
5. 阶段收口或对外同步前，优先执行 `npm.cmd run check:proof:strict` 保留双命令证据

## 2. 命名规则

当前高频命令统一使用 `task:*` 前缀：

1. `task:init:*`
   - 首次初始化、环境准备
2. `task:start:*`
   - 启动前端 / 后端 / 完整程序；需要兼顾效率时使用 `task:start:fast` 或 `task:start:python:lite`
3. `task:access:*`
   - 权限、凭据、secret、配置同步
4. `task:verify:*`
   - 检查、测试、验收
5. `task:data:*`
   - seed、同步、数据准备

## 4. 边界提醒

以下事项不伪装成“仓库内可完全自动完成”：

1. Cloudflare 账号登录本身
2. 新建或申请云端 Token
3. 生产 secret 的值来源决策
4. 真实远端发布

这些任务可以有标准命令入口，但仍需满足外部前提。
