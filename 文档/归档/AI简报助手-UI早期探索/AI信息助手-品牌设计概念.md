## 六、使用指南

### 6.1 Logo使用规范

1. **最小尺寸**：Logo最小显示尺寸为 120×60px
2. **安全区域**：Logo周围保留至少 20px 的安全区域
3. **背景要求**：确保Logo与背景有足够对比度
4. **禁止变形**：禁止拉伸、压缩或旋转Logo

### 6.2 图标使用规范

1. **颜色继承**：图标使用 `currentColor`，继承父元素颜色
2. **尺寸建议**：标准尺寸 24px，可缩放至 16px-48px
3. **描边一致**：保持 1.5px 描边宽度
4. **视觉平衡**：简单图标适当放大，复杂图标适当缩小

### 6.3 标签使用场景

| 标签 | 使用场景 |
|------|---------|
| AI智能驱动 | 产品介绍页、功能说明、营销材料 |
| 成长记录 | 用户故事页、周报月报、数据展示 |
| 信息收集 | 收藏管理页、推送通知、设置页 |
| 极简操作 | 新手引导、功能介绍、对比优势 |

## 七、用户引导策略

### 7.1 问题背景

产品的核心模式为"用户提出要求 → APP执行"，但当用户问"我该做什么"时，APP需要具备主动引导能力。

### 7.2 基于用户状态的差异化响应

| 用户状态 | 响应策略 | 示例 |
|---------|---------|------|
| **新用户（无数据）** | 引导建立首个目标 | "我可以帮你做这些事：信息追踪、待办提醒、想法记录" |
| **有待办事项** | 提醒待办，鼓励行动 | "今天有2个待办，建议优先处理简历投递" |
| **有关注内容** | 推送新内容，延续关注 | "今天收集了3条新内容，有感兴趣的吗？" |
| **近期不活跃** | 唤醒记忆，重新连接 | "上次你关注了AI技术，这周有5条新内容" |

### 7.3 主动建议机制

APP不只是被动响应，还可基于用户数据主动给出成长建议：

```
用户：我该做什么？

[AI] 基于你最近的关注，我有几个建议：

     💡 你关注了很多AI相关内容
        → 要不要深入了解某个方向？
     
     💡 你收藏了3个兼职机会但还没行动
        → 要不要我帮你规划投递计划？
     
     💡 你已经连续7天记录想法了
        → 今天要不要回顾一下这周的收获？
```

### 7.4 引导设计原则

| 原则 | 说明 |
|------|------|
| **不空谈** | 永远给出具体选项，不说"你想做什么都行" |
| **有依据** | 基于用户历史数据给出建议 |
| **可行动** | 每个建议都有明确的下一步 |
| **渐进式** | 从简单到复杂，逐步引导 |

### 7.5 引导对话示例

#### 场景A：新用户（无任何数据）

```
用户：我该做什么？

[AI] 你好！看来我们刚开始认识 😊

     我可以帮你做这些事：
     
     📰 信息追踪 —— 告诉我你关心的领域，我帮你收集最新动态
     📝 待办提醒 —— 告诉我你的计划，我帮你记住并提醒
     📖 想法记录 —— 随时告诉我你的想法，我帮你整理
     
     你想从哪个开始？
```

#### 场景B：有待办事项的用户

```
用户：我该做什么？

[AI] 看了看你的待办，今天有这些事：

     ☐ 投递简历（截止：明天）
     ☐ 完成项目报告（截止：后天）
     
     建议优先处理简历投递，只剩1天了！
```

#### 场景C：有关注内容但无待办的用户

```
用户：我该做什么？

[AI] 今天为你收集了3条新内容：

     📰 OpenAI发布GPT-5技术预览版
     💼 远程运营岗位（200-300/天）
     📚 高效学习方法论
     
     有感兴趣的吗？
```

#### 场景D：有历史行为但近期不活跃的用户

```
用户：我该做什么？

[AI] 有一阵子没见了！

     上次你关注了「AI技术」和「远程工作」
     这周有5条新内容等你查看
     
     或者，有什么新的想做的事吗？
```

## 八、功能拓展规划

### 8.1 MVP阶段纳入功能

| 功能 | 描述 | 纳入理由 |
|------|------|---------|
| **智能摘要** | 多条内容合并为摘要，解决信息过载 | 核心痛点，AI差异化优势 |
| **每日精选** | 只推送最重要的3条内容 | 核心痛点，实现简单 |
| **成就系统** | 徽章、里程碑、成长等级 | 增强用户粘性，契合"见证成长"定位 |

### 8.2 后续迭代功能

| 功能 | 描述 | 纳入理由 |
|------|------|---------|
| **目标追踪** | 设定目标、进度可视化 | 增强成长感知 |
| **打卡日历** | 可视化打卡记录 | 可视化成长轨迹 |
| **分享功能** | 生成分享链接、分享到社交平台 | 增加产品传播性 |

### 8.3 成就系统设计

#### 8.3.1 徽章体系

| 徽章类型 | 徽章名称 | 获得条件 | 图标建议 |
|---------|---------|---------|---------|
| **入门徽章** | 新手上路 | 完成首次关注 | 🌱 |
| **入门徽章** | 记录者 | 完成首次记录 | ✏️ |
| **入门徽章** | 行动派 | 完成首个待办 | ⚡ |
| **坚持徽章** | 连续7天 | 连续使用7天 | 🔥 |
| **坚持徽章** | 连续30天 | 连续使用30天 | 💪 |
| **坚持徽章** | 连续100天 | 连续使用100天 | 🏆 |
| **成长徽章** | 收藏达人 | 收藏100条内容 | 📚 |
| **成长徽章** | 阅读达人 | 阅读50条内容 | 👀 |
| **成长徽章** | 记录达人 | 记录100条想法 | 📝 |

#### 8.3.2 成长等级

| 等级 | 名称 | 所需经验 | 特权 |
|------|------|---------|------|
| Lv.1 | 种子 | 0 | 基础功能 |
| Lv.2 | 幼苗 | 100 | 解锁自定义标签 |
| Lv.3 | 小树 | 500 | 解锁高级导出 |
| Lv.4 | 大树 | 1500 | 解锁主题切换 |
| Lv.5 | 森林 | 5000 | 解锁专属徽章 |

#### 8.3.3 经验获取规则

| 行为 | 经验值 | 每日上限 |
|------|--------|---------|
| 每日登录 | +5 | 5 |
| 收藏内容 | +2 | 20 |
| 阅读内容 | +1 | 10 |
| 记录想法 | +3 | 30 |
| 完成待办 | +5 | 25 |
| 连续打卡 | +10 | 10 |

### 8.4 功能适配度评估标准

| 评估维度 | 权重 | 说明 |
|---------|------|------|
| 定位契合度 | 30% | 是否符合"收集、记录、成长"核心定位 |
| 价值贡献度 | 25% | 是否增强核心价值主张 |
| 实现复杂度 | 20% | 开发成本和技术难度 |
| 用户需求度 | 15% | 用户是否真正需要 |
| 差异化价值 | 10% | 是否形成产品差异化 |
