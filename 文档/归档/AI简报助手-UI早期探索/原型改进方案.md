## 一、改进概述

### 1.1 核心问题

| 问题 | 改进方向 |
|------|----------|
| 底部导航栏与对话式界面冲突 | 改用侧边栏导航 |
| 首页对话式设计学习成本高 | 对话内嵌快捷入口 |
| 机会详情缺少决策信息 | 增加评估卡片 |
| 观点表达交互逻辑模糊 | 两步式表达流程 |
| 缺少"后悔机制" | 添加暂停/放弃功能 |
| 画像页缺少行动引导 | 增加行动建议区 |

### 1.2 设计规范引用

> **设计细节（颜色、字体、间距、组件样式等）全部参考 [页面设计文档.md](file:///e:/python/杂谈-想法记录与实践/页面设计文档.md)**

本方案仅定义架构层面的改动。

## 二、导航架构变更

### 2.1 变更内容

| 项目 | 原设计 | 改进后 |
|------|--------|--------|
| **导航方式** | 底部Tab导航 | 侧边栏导航 |
| **主界面** | 首页（对话式）+ 4个Tab | 纯对话主界面 |
| **导航入口** | 底部固定 | 顶部菜单按钮（≡） |

### 2.2 导航结构

```
主界面（默认）
│
├─ 顶部栏：≡ 菜单按钮 | Logo | ⚙️ 设置
│
├─ 对话区域（沉浸式）
│
└─ 底部输入区：输入框 + 快捷标签

点击 ≡ 展开侧边栏：
│
├─ 🔍 发现热点
├─ 🎯 成长机遇
├─ 📋 我的计划
├─ ──────────
├─ 🌟 我的画像
├─ 📊 年度报告
└─ ⚙️ 设置
```

### 2.3 变更理由

| 对比项 | 底部Tab | 侧边栏 |
|--------|---------|--------|
| 沉浸感 | 打断对话 | 完整沉浸 |
| 一致性 | 与聊天软件不符 | 符合ChatGPT等习惯 |
| 可扩展 | 最多5个 | 无限扩展 |

## 六、交互流程变更

### 6.1 新增流程

#### 暂停计划流程

```
我的计划 → 点击暂停 → 弹窗确认 → 选择原因 → 确认暂停
```

#### 放弃计划流程

```
我的计划 → 点击放弃 → 弹窗确认 → 选择原因 → 确认放弃
```

#### 继续计划流程

```
我的计划 → 已暂停Tab → 点击继续 → 恢复进度
```

### 6.2 修改流程

#### 观点表达流程

```
原流程：
热点详情 → [立场标签] + [输入框] → 提交

新流程：
热点详情 → 选择立场 → 写理由（可选）→ 提交
```

#### 机会决策流程

```
原流程：
机会详情 → [加入计划]

新流程：
机会详情 → [开始行动] 或 [先试试看]
```
