# 我的页与个人沉淀入口正式化

- 文档状态：进行中
- 建立时间：2026-04-05
- 所属阶段：阶段十三 - 下一轮业务主线收益判断
- 文档职责：承接“我的页 / 个人沉淀入口正式化”这条当前下一轮推荐业务主线的事实、边界和下一步入口

---

## 1. 为什么当前切到这条主线

阶段十三当前已经完成业务级收益排序，新的高收益真实缺口已经变成：

1. 底部导航存在“我的”入口
2. 产品功能文档也明确把“我的页”定义成：
   - 个人中心
   - 收藏
   - 画像
   - 历史日志
   - 历史简报
   - 设置
3. 但当前代码现实是：
   - `/me` 实际落到 `GrowthPage`
   - `MyPage.tsx` 只是 `GrowthPage` 的别名

因此当前下一条真正值得进入执行面的业务主线，应正式收口为：

- `我的页与个人沉淀入口正式化`

---

## 2. 当前真实现状

### 2.1 已经成立的能力

当前已经成立：

1. `GrowthPage`
2. `CollectionsPage`
3. `ProfilePage`
4. `HistoryLogsPage`
5. `HistoryBriefPage`
6. `SettingsPage / NotificationSettingsPage / HelpFeedbackPage / AboutPage`

这意味着：

- “我的页”需要组织的大多数子页面其实都已经存在

### 2.2 当前真实缺口

当前真正缺的不是更多子页，而是：

1. 没有一个真正独立的“我的页 / 个人中心页”
2. 底部导航“我的”当前直接落到成长页
3. 成长页当前更像：
   - 成长摘要
   - 报告入口
   - 历史回看入口
4. 它并不等价于真正的个人中心页

因此当前缺口不是：

- 子页面没建

而是：

- 个人沉淀入口中心没有真正产品化

### 2.3 当前这一轮实现结果

当前这一轮已经完成：

1. `MyPage.tsx` 当前已不再是 `GrowthPage` 的别名，而是独立页面
2. 路由层当前已改为：
   - `/me -> MyPage`
   - `/growth -> GrowthPage`
3. `MyPage` 当前已按“入口 + 成长摘要”收口：
   - 顶部保留最小成长摘要
   - 主体承接收藏、画像、历史日志、历史简报、设置、帮助反馈等入口
4. `GrowthPage` 当前继续保持独立成长页身份，不再默认承担“我的页”职责
5. 帮助文案当前也已同步到：
   - 从“我的页”进入历史简报与相关个人沉淀入口

---

## 3. 当前边界

这条主线当前只做：

1. `MyPage` 是否应从 `GrowthPage` 中独立出来
2. 底部导航“我的”是否应该指向真正的个人中心页
3. 个人沉淀入口如何在“我的页”稳定组织

当前明确不做：

1. 继续扩 `user_profiles`
2. 继续扩 `reports / briefings`
3. 重做成长页对象层
4. 借“我的页”之名继续扩更多设置或详情页覆盖面

---

## 4. 当前阶段性结论

当前这条主线应先收口为：

1. 下一条真正值得进入执行面的业务主线，不是继续深挖历史简报页
2. 当前更高收益的主线应正式切到：
   - `我的页与个人沉淀入口正式化`
3. 第一优先要回答的是：
   - `/me` 是否应从 `GrowthPage` 切回真正的 `MyPage`
4. 当前这个第一优先问题已经完成第一轮最小实现
5. 当前这条主线已经具备继续判断“是否足够形成阶段结果”的基础
6. 当前已经足够形成稳定阶段结果，可以继续向总表 / 子纲 / 总纲上推
