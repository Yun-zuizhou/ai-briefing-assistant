# 阶段十一候选主线收益判断

- 文档状态：进行中
- 建立时间：2026-04-05
- 文档职责：在阶段十“主闭环页面正式读侧整合与对象消费收口”已基本收尾后，判断下一条真正值得进入执行面的主线

---

## 1. 当前要回答的问题

当前项目已经形成新的现实前提：

1. 主闭环正式读侧已具备稳定运行和稳定验证基础
2. 当前主线涉及的数据库正式承接位与关键后端接口已基本建立
3. 当前主要页面已经不再停留在“手工拼旧接口”的过渡态

在这个前提下，当前真正要回答的是：

`下一条高收益主线，是否还应该继续停留在现有后端/页面整合层。`

---

## 2. 候选主线比较

### 候选一：继续补当前页面尾差

当前现实：

1. `ActionsPage` 分组口径尾差已收口
2. `GrowthPage` 映射字段与 `TodayPage` 交互尾差当前都被判断为低收益
3. 若继续深挖，很容易进入细部打磨，而不是新的阶段主线

收益判断：

1. 当前收益偏低
2. 更像细修，不像新的项目级主线

### 候选二：重新回到对象层扩面

当前现实：

1. `reports / briefings / user_profiles / opportunity_follows` 的第一轮正式化都已收口
2. 这些子线的进一步扩面此前已多轮被判断为后置

收益判断：

1. 当前不应回退
2. 容易再次进入局部补齐

### 候选三：主线成熟度确认与下一阶段收益排序

当前现实：

1. 当前项目已经具备“后端主接口基本建立 + 页面正式读侧基本接上 + 主线测试稳定”的阶段成果
2. 但项目级口径、阶段退出动作、下一阶段真正该做什么，还没有被单独收口
3. 如果直接继续做零散实现，容易在“已稳定但未切阶段”的状态里停留过久

收益判断：

1. 当前收益最高
2. 因为这会直接决定：
   - 阶段十如何正式退出
   - 项目当前是否可以明确进入“主线稳定态”
   - 下一阶段应该优先做什么，而不是继续围绕旧主线打磨

### 候选四：二级页面正式读侧整合与会话读侧消费收口

当前现实：

1. 主闭环三页已经基本站到正式读侧上
2. 但二级页面当前仍存在明显分层差异：
   - `ProfilePage` 虽然已走 `preferences/profile`，但仍是独立画像页消费
   - `JournalPage` 仍在手工拼 `getNotes() + getHistory()`
   - `HistoryLogsPage` 仍直接读 `getHistory()`
3. `ChatPage` 当前更明显：
   - 后端正式会话读取接口 `getChatSessions / getChatSessionMessages` 已成立
   - 前端 `useChatLogic` 仍主要依赖本地 `localStorage` 会话态
4. 这意味着：
   - 主闭环已稳定
   - 但二级页面和正式会话读侧之间还没有形成统一消费面

收益判断：

1. 当前收益最高
2. 因为这条线不是继续扩对象，而是把已经成立的正式读侧继续收回到用户可感知的二级页面里
3. 其中最高优先应是：
   - `ChatPage` 是否真正切到正式会话读取链
4. 第二优先应是：
   - `JournalPage / HistoryLogsPage` 是否值得进入统一历史/记录读侧整合

---

## 3. 当前结果级结论

当前下一条真正值得进入执行面的主线，应正式收口为：

- `阶段十一 - 二级页面正式读侧整合与会话读侧消费收口`

对应判断：

1. 阶段十当前应视为已基本收尾
2. 下一步不应继续停留在阶段十页面尾差打磨
3. 阶段十一的成熟度确认已经完成第一步：
   - 当前数据库主承接位可视为主线级基本稳定
   - 当前关键后端接口可视为主线级基本建立完成
4. 在这个前提下，下一条真正高收益主线不是对象扩面，而是：
   - `ChatPage` 正式会话读侧消费
   - `JournalPage / HistoryLogsPage` 一类二级页面正式读侧整合

当前这一轮已完成的结果：

1. `ChatPage` 正式会话读侧消费收口已完成第一轮最小实现
2. `useChatLogic` 当前已优先读取最近一个正式 session，并在刷新后优先从正式会话读取链恢复消息
3. 本地 `localStorage` 已从主会话事实源降级为缓存兜底
4. `JournalPage` 已开始从手工拼 `getNotes() + getHistory()` 转向 `getNotes() + getGrowthOverview()`
5. `HistoryLogsPage` 当前维持 `getHistory()`，并明确不借此继续扩新接口

因此阶段十一当前应进一步收口为：

- `阶段十一的核心实现工作已经完成，当前可以进入收尾与下一阶段主线判断`

阶段十一当前最合适的下一步入口，不再是继续补这轮实现，而是：
