## 2. 当前真实现状

### 2.1 已经成立的能力

当前已成立：

1. `GET /api/v1/reports`
2. `GET /api/v1/reports/weekly`
3. `GET /api/v1/reports/monthly`
4. `GET /api/v1/reports/annual`
5. `WeeklyReportPage / MonthlyReportPage / AnnualReportPage` 已接正式接口

这意味着：

- 周期回顾详情页读侧已经基本成立

### 2.2 当前真实缺口

当前前端仍存在明确缺口：

1. `HistoryBriefPage` 仍是整页 mock
2. 虽然前端已有 `getReports()`，但历史简报页还没有真正消费它
3. 当前“历史简报 -> 对应周期报告详情”的回看链路还没有正式成立

因此当前缺口不是：

- 报告接口未建立

而是：

- 报告历史入口和历史回看产品面还未正式消费已成立接口

### 2.3 当前这一轮实现结果

当前这一轮已经完成：

1. `HistoryBriefPage` 已改为直接调用：
   - `getReports()`
2. 页面当前不再伪造“14 天历史简报”随机列表
3. 页面当前已直接消费：
   - `reportType`
   - `reportTitle`
   - `generatedAt`
   - `available`
4. 可用报告当前已直接跳转：
   - `weekly -> /weekly-report`
   - `monthly -> /monthly-report`
   - `annual -> /annual-report`
5. 不可用报告当前显示禁用态与“暂未生成”说明
6. 原 fake detail modal、fake export / share modal 当前已移除，历史页当前正式收口为“周期回顾回看入口页”

### 2.4 当前第二轮产品面收口

当前第二轮已完成：

1. 帮助页中“如何查看历史简报”的产品文案已对齐到当前真实链路
2. 主线测试当前已显式补上：
   - `GET /api/v1/reports`
3. 这意味着当前不仅页面接上了正式读侧，而且：
   - 报告入口接口已进入主线级显式验收覆盖
   - 用户帮助口径也不再停留在旧产品理解上

### 2.5 当前第三轮入口链路收口

当前第三轮已完成：

1. `GrowthPage` 当前已补上：
   - Masthead 中的“简报”入口
   - 报告区块中的“查看历史”入口
2. 这意味着当前不只是：
   - `HistoryBriefPage` 本身已接到 `getReports()`
3. 而且用户当前也可以从成长页稳定进入：
   - `历史简报 -> 周 / 月 / 年报详情`

## 3. 当前边界

这条主线当前只做：

1. `HistoryBriefPage` 是否切到 `getReports()`
2. 周报 / 月报 / 年报入口是否在历史简报页形成稳定回看链
3. 历史简报页从 mock 转向正式报告列表读侧

当前明确不做：

1. 重新扩 `reports` 对象层
2. 重新设计周报 / 月报 / 年报详情结构
3. 重拉 `briefings` 历史对象扩面
4. 以历史简报页为入口继续扩分享图片、复杂导出链路
