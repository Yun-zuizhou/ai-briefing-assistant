## 1. 本轮要解决的问题

上一轮推进时，已经暴露出一个工程口径问题：

1. 文档和协作提示词里默认把前端验收写成 `npm.cmd run build` / `npm.cmd run lint`
2. 但根目录 `package.json` 实际并没有 `build` / `lint` 脚本
3. 真正可运行的命令仍然要进入 `prototype/` 目录

这会直接导致：

1. 新对话里的 Agent 很容易沿用错误口径
2. 用户按文档执行时会报“Missing script”
3. 工程入口和文档入口再次脱节
