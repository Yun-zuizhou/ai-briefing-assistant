# favorites内容归档纠偏子线判断

- 文档状态：进行中
- 建立时间：2026-04-04
- 所属阶段：阶段二 - Today / 内容层设计与事实层收束
- 作用：明确 `favorites` 是否应继续纳入 Chat 纠偏主线，避免后续在“会话分流”与“内容归档整理”之间反复摇摆

---

## 1. 当前判断

当前不建议把 `favorites` 纳入 Chat 的 `reclassify` 主线。

这不是说收藏不重要，而是它属于另一类问题：

`内容归档纠偏`

而不是：

`会话分流纠偏`

---

## 2. 为什么不并进当前 Chat 主线

原因有三点：

1. `favorites` 对应的是内容收藏关系，不是 Chat 意图落库结果本体
2. 收藏更接近统一内容层、内容归档和后续整理策略
3. 如果现在硬并进 Chat `reclassify`，会把“会话分流纠偏”和“内容归档整理”两条线混成一条

因此，当前更合理的做法是：

1. Chat 主线继续围绕 `todo / note / history`
2. `favorites` 单独作为“内容归档纠偏”子线后续判断

---

## 3. 当前对子线的最小定义

`favorites` 子线后续主要回答三件事：

1. 收藏对象是否需要重新归类到别的内容事实
2. 收藏和统一内容层实体化后如何关联
3. 收藏记录是否需要补“为什么收藏 / 是否仍 relevant”这类整理能力

这三件事都不应在当前 Chat `reclassify` 接口里草率处理。

---

## 4. 当前正确口径

当前应更新为：

1. Chat 最小纠偏主线已覆盖 `todo / note / history`
2. `favorites` 暂不进入 Chat 纠偏主线
3. 如后续需要推进 `favorites`，应单独作为“内容归档纠偏”子线立项判断
