# Chat后端纠偏留口最小落地

- 文档状态：进行中
- 建立时间：2026-04-03
- 所属阶段：阶段二 - Today / 内容层设计与事实层收束
- 作用：记录 Chat 在“事后纠偏入口最小接入”之后，继续把后端正式纠偏留口落到可调用、可回归状态

---

## 1. 本轮目标

本轮不是一次性完成完整回滚系统，而是把：

`前端纠偏入口 -> 后端 reclassify -> 新结果回到消息流`

这条最小主链先真实跑通。

---

## 2. 本轮已完成

### 2.1 后端已新增最小 `reclassify` 入口

当前在：

- [chat.py](/E:/python/杂谈-想法记录与实践/app/api/v1/chat.py)

已新增：

- `/api/v1/chat/reclassify`

当前最小支持范围：

1. `todo -> record_thought / fragmented_thought / chat_only`
2. `note -> create_todo / record_thought / fragmented_thought / chat_only`

### 2.2 旧对象处理策略已先明确

当前先按最小真实策略落地：

1. `todo -> note`
   - 创建新记录
   - 原待办改为 `cancelled`
2. `todo -> chat_only`
   - 原待办改为 `cancelled`
   - 不再保留新的结构化对象
3. `note -> todo`
   - 创建新待办
   - 原记录保留
4. `note -> record_thought / fragmented_thought`
   - 原记录保留
   - 按新的理解方式更新最小标签
5. `note -> chat_only`
   - 原记录保留
   - 当前只调整理解口径，不删除历史写入

这意味着当前已经不再停留在“以后再说旧对象怎么处理”，而是有了第一版真实口径。

### 2.3 前端消息流已可直连后端纠偏

当前在：

- [useChatLogic.ts](/E:/python/杂谈-想法记录与实践/prototype/src/hooks/useChatLogic.ts)
- [ChatPage.tsx](/E:/python/杂谈-想法记录与实践/prototype/src/pages/ChatPage.tsx)
- [api.ts](/E:/python/杂谈-想法记录与实践/prototype/src/services/api.ts)

已完成：

1. 消息动作可带 `correctionFrom`
2. 满足后端纠偏条件时，前端不再只做切模式，而是直接调用 `reclassify`
3. 纠偏结果会重新写回消息流
4. 无法直连后端时，才回退到“带回输入框 + 切轻模式”的前端入口
