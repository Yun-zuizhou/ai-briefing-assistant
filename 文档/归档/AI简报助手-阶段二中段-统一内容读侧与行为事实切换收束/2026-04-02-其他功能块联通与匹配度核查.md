# 其他功能块联通与匹配度核查

- 文档状态：进行中
- 建立时间：2026-04-02
- 所属阶段：阶段 4 主闭环联调前置核查
- 核查范围：除 Today 页、对话页外的其他主要功能块
- 核查目标：
  - 判断当前各功能块是否真正联通
  - 判断当前实现是否与 `项目功能文档`、`项目核心总纲` 前半段要求匹配
  - 先稳住旁支，再进入 Today / 对话页的困难检修

---

## 1. 本轮结论

本轮可以明确下结论：

当前项目除 Today 页、对话页外的其他功能块，已经出现明显分层：

### 第一类：已基本接入真实后端的功能块

1. 行动页（ActionsPage）
2. 成长页（GrowthPage）
3. 对应的基础读接口：
   - `/api/v1/todos`
   - `/api/v1/favorites`
   - `/api/v1/notes`
   - `/api/v1/history`

### 第二类：页面存在，但当前仍主要停留在本地 mock / context / 演示态

1. 日志页（JournalPage）
2. 热点页（HotTopicsPage）
3. 收藏页（CollectionsPage）
4. 用户画像页（ProfilePage）
5. 历史日志页（HistoryLogsPage）
6. 周报 / 月报 / 年报
7. 文章详情页（ArticlePage）

因此当前不能把“其他功能块都已正确联通”作为既成事实。

更准确的判断是：

`行动 / 成长已进入真实接口过渡态，但日志、热点、收藏、画像、历史、报告等大量页面仍主要停留在本地 mock 或 context 层。`

---

## 2. 与项目功能文档、项目核心总纲前半段的匹配判断

### 2.1 匹配的部分

以下方向与总纲前半段目标总体一致：

1. 项目仍围绕“今日信息 -> 行动转化 -> 自我记录 -> 周期回顾”主闭环推进
2. 行动页已经在向“统一行动中心”收束
3. 成长页已经在向“成长中心”收束
4. 基础事实层已经具备 `todo / favorite / note / history` 这几类真实数据入口

### 2.2 不匹配或弱匹配的部分

以下部分与 `项目功能文档` 或 `项目核心总纲` 前半段仍存在偏差：

1. 热点追踪页仍主要读取本地 mock，而不是实时热点事实层
2. 收藏页仍主要读取本地 mock，而不是 `/favorites` 真实接口
3. 日志页仍主要读取本地 thoughtRecords / mock story，而不是 `/notes` 与 `/history`
4. 用户画像页仍主要读取本地 context，而不是基于真实行为数据生成
5. 历史日志页仍主要读取本地 context thoughtRecords，而不是 `/history`
6. 周报 / 月报 / 年报仍是展示型 mock 报告，不是基于真实数据的回顾体系
7. 文章详情页仍是路由 state 驱动，不是从真实内容接口读取

因此，当前“项目功能文档”里列出的不少功能，页面虽然存在，但并没有全部进入真实联通态。

## 3. 分模块核查结果

## 3.1 行动页（ActionsPage）

### 当前状态

已基本进入真实接口过渡态。

### 已成立

1. 前端实际请求：
   - `/api/v1/todos`
   - `/api/v1/favorites`
2. 待办切换完成状态会调用真实更新接口
3. 删除待办会调用真实删除接口

### 当前问题

1. 收藏区虽走 `/favorites`，但仍缺“从热点 / 文章页真实写入收藏”这一端到端闭环的统一收束
2. 页面仍保留本地打卡状态 `localStorage`，未进入统一事实层

### 判断

行动页是当前除 Today / 对话外，联通程度最高的页面之一。

---

## 3.2 成长页（GrowthPage）

### 当前状态

已进入真实接口过渡态，但仍混有本地 context。

### 已成立

1. 前端实际请求：
   - `/api/v1/notes`
   - `/api/v1/history`
2. 页面可展示真实记录数量和历史痕迹数量

### 当前问题

1. 活跃兴趣仍来自 `AppContext`
2. 一句话画像仍是页面内静态描述
3. 报告入口仍未接到真实生成链路

### 判断

成长页比日志页更接近真实态，但还不是完整真实成长中心。

## 5. 当前优先级判断

基于这轮核查，当前最合理的优先级应当是：

1. 先确认“其他功能块里哪些已经真实联通，哪些只是演示态”
2. 把行动页、成长页认定为当前较稳定的过渡态基础
3. 把日志、热点、收藏、画像、历史、报告明确认定为待收束页
4. 在这个基础上再进入 Today 页、对话页的困难检修

这意味着：

当前不应该直接假设“除了 Today / 对话之外都已稳”。

更准确的说法应该是：

`真正比较接近真实联通的，是行动页和成长页；其他很多功能块仍只是页面存在，并未完成真实联通。`

---

## 6. 对下一步的建议

在进入 Today 页、对话页深入检修前，最合理的预处理动作是：

1. 先把“其他功能块”的状态明确标为：
   - 已进入真实接口过渡态
   - 仍停留在本地 mock / context 层
2. 不要求这些旁支先全部真实化
3. 但至少要避免把这些页面误判为已经联通完成

然后再进入更难的两个核心块：

1. Today 页：信息获取、信息加工、信息整合、信息呈现
2. 对话页：意图识别、执行结果、系统回写、跳转建议
