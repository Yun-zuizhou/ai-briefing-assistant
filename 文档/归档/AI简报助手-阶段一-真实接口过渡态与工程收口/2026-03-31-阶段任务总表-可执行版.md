目标：

- 对话记录想法后，成长页或日志页可见

验收标准：

- 新记录能出现在成长体系内
- 成长页或日志页刷新后仍可见
- 重新进入应用后仍可查询到该记录
- 记录来源为真实接口返回，而非仅本地状态保留

状态：已完成第一轮验证（共享虚拟状态下写后读一致已成立，真实持久化未完成）

### T4-3 对话调整关注回写

目标：

- 对话调整关注后，今日页推荐内容变化

验收标准：

- 推荐理由和内容与新关注一致
- 今日页刷新后推荐结果保持一致
- 重新进入应用后推荐逻辑仍基于最新关注生效
- 推荐结果来自真实聚合接口，而非静态 mock 切换

状态：进行中（代码断点已修正，待接口级验证）

### T4-4 跑通主闭环验证

目标：

- 以真实状态验证“信息 -> 行动 -> 记录 -> 回顾”

验收标准：

至少能跑通以下链路之一：

1. 今日看到内容 -> 转待办 -> 行动页可见 -> 成长页有痕迹
2. 对话输入想法 -> 成长页可见 -> 周期回顾可引用

补充验证要求：

- 链路中每一步均由真实接口读写支撑
- 至少进行一次页面刷新验证
- 至少进行一次应用重新进入后的持久化验证
- 不允许通过仅修改前端本地状态来判定联调成功

状态：进行中（局部链路已成立，完整持久化闭环未完成）

## 13. 下一步

当前最合理的下一步不是直接继续扩大 Today 的困难改造，而是先把“所有主要页面至少进入真实接口过渡态”这件事拆成可执行顺序：

1. 先把已有表已支撑的页面改为真实接口主驱动：
   - `CollectionsPage -> /favorites`
   - `HistoryLogsPage -> /history`
   - `HotTopicsPage -> /hot-topics`
   - `JournalPage -> /notes + /history`
2. 再补缺失但高必要性的关系表与聚合表：
   - `user_interests`
   - `user_settings`
   - `opportunity_follows`
   - `user_profiles`
   - `reports`
3. 最后进入高难度收束：
   - `TodayPage -> briefings + 内容聚合层`
   - `ChatPage -> 与真实事实层统一`
   - `ArticlePage -> 统一内容引用规则或统一内容层`

当前联调结论补充：

- 读取链路已可继续推进
- 写入链路已具虚拟回退
- 共享虚拟状态下，`Chat -> Actions` 与 `Chat -> Growth` 写后读一致已成立
- `Chat -> interest -> Today` 的第一层链路验证已完成，但 Today 的内容事实层仍不真实
- 当前代码存在一个需要显式验证的断点：
  - `chat/execute` 的真实分支更新 `users.interests`
  - `/api/v1/dashboard/today` 当前读取共享虚拟兴趣状态
- 该断点的代码修正已完成：真实兴趣写入成功后，会同步镜像到共享虚拟兴趣状态
- 当前进一步确认：对全项目来说，更大的问题不只是 Today，而是大量旁支页面仍由 `mockData / AppContext` 充当主数据源
- 因此，阶段 4 与阶段 5 之间需要补一层现实动作：
  - 先让所有主要页面进入真实接口过渡态
  - 再推进 mock 从主流程退场
- 当前又已完成：
  - `reports` 独立聚合接口第一轮已落地
  - `WeeklyReportPage / MonthlyReportPage / AnnualReportPage` 已切到真实接口过渡态
  - 报告页已脱离前端 `mockData / 硬编码` 主驱动
  - 当前报告文案、趋势解释仍属于接口聚合生成的过渡态输出，不是最终正式报告事实层

这一步完成后，项目才真正进入实现重构，而不是继续停留在结构讨论。
