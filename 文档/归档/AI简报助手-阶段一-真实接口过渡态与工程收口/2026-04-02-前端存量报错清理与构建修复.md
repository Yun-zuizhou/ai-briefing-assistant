## 1. 本轮目标

清理当前会干扰主线推进的前端存量问题，并恢复 `prototype` 的“可构建 + lint 通过”状态。

---

## 2. 本轮已完成

### 2.1 已清理的存量报错

本轮清理了以下文件中的未使用导入 / 未使用变量报错：

1. `prototype/src/pages/AboutPage.tsx`
2. `prototype/src/pages/HomePage.tsx`
3. `prototype/src/pages/SettingsPage.tsx`
4. `prototype/src/utils/exportUtils.ts`

这些问题属于仓库原有存量问题，不是本轮“页面真实接口过渡态切换”引入的新错误。

### 2.2 已修复的构建命令问题

当前项目使用的是：

- `vite@8.0.0-beta.18`

在当前 Windows 环境下，默认的配置加载方式会在构建时触发：

- `spawn EPERM`

本轮已验证：

1. `vite build` 默认加载器会失败
2. `vite build --configLoader native` 可正常通过

因此已将 `prototype/package.json` 中的构建脚本调整为：

- `tsc -b && vite build --configLoader native`

### 2.3 已完成的 lint 存量清理

本轮继续完成了以下前端存量问题清理：

1. `prototype/src/components/business/HotTopicCard.tsx`
   - 去除 render 期 `Math.random`
   - 改为稳定的兜底趋势值，避免纯度规则报错

2. `prototype/src/context/AppContext.tsx`
   - 不再从该文件导出 `useAppContext`
   - 保留 `AppProvider` 为组件导出，避免 `react-refresh/only-export-components`

3. `prototype/src/hooks/useChatLogic.ts`
   - 收紧 `IntentResponse -> ParsedIntent` 的类型
   - 去除 `any`

4. `prototype/src/pages/ActionsPage.tsx`
   - 清理双重布尔取反
   - 修正筛选统计对象中的重复字段

5. `prototype/src/services/api.ts`
   - 为简报生成、AI 生成与统计接口补充最小 JSON 类型
   - 去除 `any`

6. `prototype/src/utils/exportUtils.ts`
   - 修正 `case` 块内声明问题

7. `prototype/src/utils/intentParser.ts`
   - 清理未使用变量
   - 保持测试输出逻辑完整

8. `prototype/tsconfig.app.json`
   - 补充 `resolveJsonModule`
   - 修复 `mockData.ts` 对 `mock-data/*.json` 的编译识别问题
