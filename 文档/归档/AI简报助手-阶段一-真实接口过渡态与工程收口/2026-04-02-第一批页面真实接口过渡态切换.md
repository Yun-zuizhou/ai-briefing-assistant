# 第一批页面真实接口过渡态切换

- 文档状态：进行中
- 建立时间：2026-04-02
- 所属阶段：所有主要页面进入真实接口过渡态的第一批实施
- 对应范围：
  - `CollectionsPage`
  - `HistoryLogsPage`
  - `HotTopicsPage`
  - `JournalPage`

---

## 1. 本轮已完成

本轮已按“先把已有真实表已能支撑的页面切到真实接口过渡态”的顺序，完成第一批页面改造：

1. `CollectionsPage`
   - 已从 `loadFavorites()` 切到 `/api/v1/favorites`
   - 已支持真实删除收藏
   - 不再以本地 mock 作为收藏主源

2. `HistoryLogsPage`
   - 已从 `AppContext.thoughtRecords` 切到 `/api/v1/history`
   - 已按真实接口返回的 `created_at / event_type / title / summary` 展示
   - 不再把前端 thoughtRecords 当作主源

3. `HotTopicsPage`
   - 已从本地 `hotTopics` 切到 `/api/v1/hot-topics`
   - 收藏行为已切到 `/api/v1/favorites`
   - 热点列表已进入真实接口过渡态
   - 详情弹层当前仍是过渡版，只展示真实列表字段，不再伪装成完整深度聚合详情

4. `JournalPage`
   - 已从 `loadThoughts() / generateDailyStories() / loadUserProfile()` 主驱动，切到 `/api/v1/notes + /api/v1/history`
   - 已支持真实删除记录
   - “今日叙事”“成长轨迹”“想法记录”已改为基于真实接口结果的过渡态展示

5. `apiService`
   - 已补：
     - `createFavorite`
     - `deleteFavorite`
     - `createNote`
     - `deleteNote`
   - 已修复本轮引入的 `request` 可见性问题

---

## 2. 当前状态判断

本轮完成后，可以明确说：

`CollectionsPage / HistoryLogsPage / HotTopicsPage / JournalPage 已从 context/mock 主驱动切入真实接口过渡态。`

但这里仍需保持边界清晰：

### 已进入真实接口过渡态

1. 收藏页
2. 历史日志页
3. 热点页
4. 日志页

### 仍不是正式态

1. 热点详情仍未接真实深度详情聚合
2. 日志页“今日叙事”仍是基于 notes/history 的过渡拼装，不是正式 narrative 生成链路
3. 文章详情页仍未统一到真实内容层
4. 报告体系、画像体系、兴趣与设置体系仍未完成缺失表接入

---

## 3. 当前暴露出的现实问题

本轮执行后，前端构建检查已实际运行。

### 3.1 与本轮改动直接相关的问题

已修复：

1. `api.ts` 中 `request` 可见性导致子类不可访问
2. `IntentResponse` 重复导出冲突

### 3.2 当前仍阻塞构建的项目存量问题

`npm.cmd run build` 当前仍失败，但失败原因不是本轮页面切换，而是仓库中已有的 TypeScript 未使用变量/导入问题，包括：

1. `src/pages/AboutPage.tsx`
2. `src/pages/HomePage.tsx`
3. `src/pages/SettingsPage.tsx`
4. `src/utils/exportUtils.ts`

因此本轮应作如下判断：

`第一批页面切换已完成，但前端全量构建仍被仓库原有 TS 存量问题阻塞。`
