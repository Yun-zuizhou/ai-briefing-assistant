## 1. 本次调整内容

本次完成了前端 API 层第一轮路径统一，核心改动如下：

1. 将前端主资源基础路径统一为：

`http://localhost:5000/api/v1`

2. 将 API 配置单独归为：

`http://localhost:5000/api-config`

3. 修正已接入接口的请求路径写法：

- `getHotTopics()`
- `getOpportunities()`
- `recognizeIntent()`
- 预留聚合接口：
  - `getTodayPageData()`
  - `getActionsOverview()`
  - `getGrowthOverview()`
  - `executeChat()`

4. 修正首页对列表响应结构的使用方式：

- 从直接使用 `data`
- 改为使用 `data.items`

## 2. 当前统一结果

### 2.1 资源型接口

统一规则：

- 统一走 `/api/v1`
- 方法路径不再手动重复写 `/v1`

示例：

- `/hot-topics`
- `/opportunities`
- `/intent/recognize`

### 2.2 配置型接口

统一规则：

- 单独走 `/api-config`

原因：

- 当前后端已经这样组织
- 不强行混入 `/api/v1`
