# 阶段十候选主线收益判断

- 文档状态：进行中
- 建立时间：2026-04-05
- 文档职责：在阶段九“历史回看对象组织层”第一轮最小实现已成立后，判断下一条真正值得进入执行面的主线

---

在以下子线都已阶段性收口后：

1. 阶段三内容结果层
2. 阶段四 Chat 会话正式化
3. 阶段五 `reports`
4. 阶段六 `briefings`

6. 阶段八 `opportunity_follows`

当前真正要回答的是：

`下一条高收益主线到底是谁。`

---

## 2. 候选主线比较

### 候选一：主闭环页面正式读侧整合与对象消费收口

当前现实：

1. `briefings / reports / user_profiles / opportunity_follows` 这些对象已经开始成立
2. 但很多页面当前仍是：
   - 局部即时聚合
   - 旧读侧拼装
   - 结果对象已成立但页面消费还未完全收口
3. 也就是说，对象层已经先走在前面，页面层还没有完全跟上

收益判断：

1. 当前收益最高
2. 因为项目主闭环真正面向用户的是页面，不是对象表
3. 若页面不把这些正式对象吃进去，项目会停在“对象逐个成立，但用户体验还是过渡态”的阶段

### 候选二：统一内容层重新前置

当前现实：

1. 这条线已经多轮明确“继续后置”
2. 当前 `content_ref + /api/v1/content/by-ref` 稳定过渡态已经成立

收益判断：

1. 当前收益不如页面正式读侧整合
2. 继续把它拉回主线，很容易重新陷入架构层打转

### 候选三：继续扩各条已收口子线覆盖面

当前现实：

1. `reports` 可继续扩 `annual`
2. `briefings` 可继续做历史列表
3. `user_profiles` 可继续做详情页与 diff
4. `opportunity_follows` 可继续做写入口

收益判断：

1. 这些都更像覆盖面扩展，不像新的项目级主线
2. 继续深挖任何一条，都容易再次掉回局部补齐

---

## 3. 当前结果级结论

当前下一条真正值得进入执行面的主线，应正式收口为：

- `主闭环页面正式读侧整合与对象消费收口`

对应结论：

1. 统一内容层继续后置
2. 各条已收口子线的覆盖面扩展继续后置
3. 阶段十名称可先收口为：
   - `阶段十 - 主闭环页面正式读侧整合与对象消费收口`

---

## 4. 现在离主线目标还有多远

如果把项目主线目标理解为：

`今日信息 -> 转化行动 -> 自我记录 -> 周期回顾 -> 看见成长`

当前判断是：

- **主闭环骨架大约已完成 70% 到 80%**

已经成立的部分：

1. 今日结果对象已开始成立
2. 行动转化与后续跟进已有最小对象
3. 自我记录与历史痕迹已成立
4. 周期回顾对象已开始成立
5. 成长画像对象已开始成立

仍然差的关键不是“有没有对象”，而是：

1. 页面是否已经全面切到这些正式对象
2. 页面之间是否已经形成统一、稳定、对用户可感知的正式读侧体验
3. 若对象层和页面层继续脱节，项目就仍然会有明显的“过渡态感”

所以当前距离主线目标，已经不是“差很多基础能力”，而是：

- **差最后一层页面级正式整合与收口**

---

## 5. 一句话结论

`阶段九之后，新的高收益主线不该再回到统一内容层或局部覆盖面扩展，而应切到“主闭环页面正式读侧整合与对象消费收口”；当前离项目主线目标大致还差最后 20% 到 30%，主要差在页面层而不是对象层。`
