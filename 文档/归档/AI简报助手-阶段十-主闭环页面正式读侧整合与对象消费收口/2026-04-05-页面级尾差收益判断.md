# 页面级尾差收益判断

- 文档状态：进行中
- 建立时间：2026-04-05
- 所属阶段：阶段十 - 主闭环页面正式读侧整合与对象消费收口
- 文档职责：在首页、行动页、成长页已完成第一轮正式读侧切换后，判断剩余页面级尾差是否值得继续进入执行面

---

## 1. 为什么当前切到这条主线

阶段十这一轮推进后，当前现实已经变成：

1. `TodayPage` 已基本站到 `dashboard/today`
2. `GrowthPage` 已切到 `growth-overview`
3. `ActionsPage` 已切到 `actions/overview`

也就是说，当前主闭环页面的主要矛盾，已经不再是“页面还在手工拼旧接口”，而是：

- 页面切到正式读侧后，仍残留哪些局部尾差
- 这些尾差到底只是可接受映射，还是下一轮值得继续收口的缺口

因此当前需要单独开一份并列主线文档，专门承接“页面级尾差收益判断”，而不是继续把这些尾差判断堆进对象消费边界文档。

---

## 2. 当前真实现状

### 2.1 ActionsPage 分组口径

当前现实：

1. 页面已经切到 `getActionsOverview()`
2. 当前正式对象直接提供的是：
   - `todayTodos`
   - `savedForLater`
   - `followingItems`
   - `reminderSummary`
3. 原页面中保留了：
   - `今日 / 未来 / 已完成` 三段筛选
4. 但当前“未来 / 已完成”并不是正式对象直接给出的分组口径，而是基于 `todayTodos` 当前返回集合做本地视图

当前进展：

1. `actions/overview` 已补齐：
   - `futureTodos`
   - `completedTodos`
2. `ActionsPage.tsx` 当前已直接消费：
   - `todayTodos`
   - `futureTodos`
   - `completedTodos`
3. 原页面中的 `今日 / 未来 / 已完成` 三段筛选，已经不再依赖页面层自行从 `todayTodos` 推断

当前判断：

- 这条尾差已经完成收口
- `ActionsPage` 当前已具备完整、稳定的正式分组读侧

### 2.2 GrowthPage 映射字段

当前现实：

1. 页面已经切到 `getGrowthOverview()`
2. 当前正式对象已直接承接：
   - `weeklySummary`
   - `keywords`
   - `persona`
   - `recentHistoryItems`
   - `reports`
3. 但原页面里还有两类展示语义并非正式对象直接提供：
   - “活跃关注项”
   - “历史总量”
4. 当前页面做法是：
   - 用 `keywords` 映射承接“活跃关注项”
   - 用 `recentHistoryItems.length` 承接“近期回看数量”

当前判断：

- 当前页面已经站到正式读侧
- 但仍存在“页面层推断字段”而不是“正式对象直接给值”的尾差
- 结合当前页面展示效果，这条尾差暂不值得继续为它补对象字段

### 2.3 TodayPage 交互尾差

当前现实：

1. 页面已经切到 `getTodayPageData()`
2. 今日总述、推荐内容、值得知道、值得行动等核心展示都已经走正式读侧
3. 当前剩余问题主要不是对象消费，而是：
   - 交互组织是否已经足够稳定
   - “记录一个想法 / 去行动 / 查看全部”等跳转是否已经形成稳定主舞台节奏
   - 页面里仍有哪些“聚合页感”而非“正式结果页感”的细部体验问题

当前判断：

- 首页当前已经不是对象消费缺口
- 但仍可能存在少量页面交互尾差
- 结合当前阶段目标，这类交互尾差当前收益偏低，继续后置更合适

---

## 3. 当前边界

这份文档当前只做“收益判断”，不直接自动转成实现任务。

当前明确不做：

1. 不为了解决尾差而重新拉起统一内容层
2. 不借尾差之名继续扩 `reports / briefings / user_profiles / opportunity_follows`
3. 不在没有明确收益前，重新设计 `actions/overview` 或 `growth-overview` 的完整对象结构
4. 不把“页面有一点映射”误判为必须立刻补对象字段

---

## 4. 当前阶段性结论

当前页面级尾差的收益判断，现应更新为：

1. `ActionsPage` 分组口径尾差已完成收口
2. `GrowthPage` 映射字段尾差当前不进入实现
   - 现有映射没有明显破坏页面语义
   - 为它新增对象字段的收益，低于当前阶段继续收尾的收益
3. `TodayPage` 交互尾差当前继续后置
   - 它更像体验细修，不是阶段十当前必须继续推进的主链缺口

当前收口结论：

1. 页面级尾差里，唯一值得立即进入实现的 `ActionsPage` 分组口径已经完成
2. `GrowthPage` 与 `TodayPage` 当前都没有继续实现的高收益理由
3. 阶段十这条“页面级尾差收益判断”子主线已经具备收尾条件
