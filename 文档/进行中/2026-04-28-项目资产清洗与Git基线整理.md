# 2026-04-28-项目资产清洗与Git基线整理

- 建立日期：2026-04-28
- 当前阶段：阶段十五尾差清理 → 阶段十六准备
- 触发原因：用户反馈"项目很脏，不敢清洗"；实际检查发现核心代码全部 untracked，Git 基线严重落后于真实工作状态

## 1. 当前真实状态（不可跳过）

经过本轮检查，发现比预期更严重的 Git 问题：

| 状态 | 文件范围 | 说明 |
|------|----------|------|
| **M（已修改）** | `app/` 下 20+ 个 Python 文件、`package.json`、`.gitignore` | 旧 Railway 时代代码的修改版 |
| **D（已删除）** | `cloudflare/d1/*`（16个文件）、`prototype/*`（60+个文件）、`Procfile` | 旧 Railway 结构，已物理删除但未提交 |
| **??（未跟踪）** | `apps/`、`infra/`、`packages/`、`tools/`、`e2e/`、`文档/`、`资产整理工作台/` 等 | **项目核心代码，从未进入 Git** |
| **Git 历史** | 仅 2 个 commit（Railway 部署相关） | 无法反映任何真实开发历史 |
| **远程仓库** | `https://github.com/Yun-zuizhou/ai-briefing-assistant.git` | 仅包含 Railway 旧代码 |

核心问题：**当前项目的 80% 以上代码不在 Git 保护范围内。一次误操作就可能丢失全部工作。**

## 2. 目标

1. 将全部正式主线代码纳入 Git 保护
2. 形成一条干净的 Git 基线，反映真实项目状态
3. 推送到 GitHub 私人仓库作为安全网
4. 清理 Git 中的旧结构残留（`prototype/`、`cloudflare/d1/` 删除确认）
5. 清洗完成后 `npm.cmd run check` 仍然全绿

## 3. 不可变更的原则

1. **不移动任何正式源码目录**：`apps/web`、`apps/edge-worker`、`packages/contracts`、`infra/cloudflare/d1`、`tools/scripts` 位置不变
2. **不修改任何代码逻辑**：本轮只做 Git 提交和 `.gitignore` 调整
3. **每步可回退**：每一步独立提交，出错可 `git reset` 回到上一步
4. **check 门槛不降级**：每一步之后验证 `npm.cmd run check` 通过

## 4. 执行步骤

### 第 0 步：安全存档（当前仓库状态快照）

将当前混乱但可运行的状态作为一个 snapshot commit 提交，确保任何操作都可回退。

```bash
git add -A
git commit -m "snapshot: 2026-04-28 清洗前安全存档 - 包含全部未跟踪代码"
```

### 第 1 步：向 GitHub 推送安全网

```bash
git push -u origin main
```

如果推送失败（远程有冲突或历史不兼容），使用 force push 但必须先确认远程没有需要保留的内容。

### 第 2 步：确认 .gitignore 覆盖完整

检查 `.gitignore` 是否覆盖了所有不应进入版本控制的文件：
- `var/local/`、`var/logs/`、`var/tmp/` — 运行产物
- `.pydeps/`、`.pydeps_runtime/` — Python 依赖缓存
- `node_modules/` — Node 依赖
- `_archive_workspace/` — 归档区
- `.env`、`apps/web/.env`、`apps/edge-worker/.dev.vars` — 敏感配置

### 第 3 步：补充忽略规则（如需要）

如果发现仍有运行产物、缓存文件或敏感配置未在 `.gitignore` 中，补充规则并单独提交。

### 第 4 步：标记清理完成，验证

```bash
npm.cmd run check
```

确认全绿后，提交最终状态标记。

## 5. 风险与缓解

| 风险 | 缓解 |
|------|------|
| `git add -A` 误提交敏感文件（.env、token） | 先检查 .gitignore，确认 .env 系列已忽略 |
| 远程仓库有他人提交导致 push 冲突 | 远程是私人仓库，无协作者，概率极低 |
| 大文件（generated seed SQL）导致仓库膨胀 | 检查 `infra/cloudflare/d1/seeds/generated` 大小，若超 10MB 考虑不入版本 |
| `_archive_workspace/` 体积巨大 | 已在 .gitignore 中，确认不会被提交 |
