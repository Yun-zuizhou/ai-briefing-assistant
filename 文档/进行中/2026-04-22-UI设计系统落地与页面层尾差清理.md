## 1. 为什么切到这条主线

当前阶段十五的核心目标是"页面级尾差清理与人工验收"。在已完成研发态文案清理、正式鉴权链路收口后，UI层存在以下阻碍人工验收的结构性问题：

1. **尺寸体系异常**：932px硬编码、字号压缩、间距不足，导致目标移动设备上的正式体验不稳定
2. **组件库虚设**：Button/Card/Tag定义了但页面未使用，717处内联style分散维护
3. **三套样式并行**：CSS类 + Tailwind + inline style无统一规范
4. **装饰过度**：Masthead占比过大、装饰性角落、噪点纹理消耗空间与性能
5. **使用逻辑问题**：TabBar顺序、开发者文案、假状态栏等不符合产品定位

这些问题直接影响"核心页面人工验收与截图证据"的质量，必须在验收前完成最小收口。

## 2. 当前真实现状

### 2.1 问题确认

| 问题类别 | 严重程度 | 证据 | 状态 |
|---------|---------|------|------|
| 硬编码尺寸 | 高 | `index.css` #root height:932px | 待修复 |
| 字号压缩 | 高 | design-tokens.ts base:0.75rem(12px) | 待修复 |
| 组件库虚设 | 高 | 25页面/9组件导出/191处CSS类 | 待修复 |
| 三套样式并行 | 高 | CSS类+Tailwind+inline style并存 | 待修复 |
| Masthead过高 | 中 | ornaments+meta占纵向140-160px | 待修复 |
| 内联样式泛滥 | 高 | 717处/25文件 | 待修复 |
| 圆角/阴影不统一 | 中 | CSS中border-radius:0 vs Card.tsx中rounded-lg | 待修复 |
| TabBar顺序 | 中 | 对话→简报→待办→日志→我的 | 待修复 |
| 开发者文案 | 中 | ChatPage空状态"这个页面的作用不是浏览内容" | 待修复 |
| 热点页重复 | 低 | 条形图+卡片列表展示同一批数据 | 待修复 |
| 可访问性不足 | 中 | 对比度2.7:1、缺少aria-label | 待修复 |
| 无骨架屏 | 低 | 全部页面loading态为文字 | 待修复 |

### 2.2 已成立的决策

1. **复古报纸风格保留**：直角(border-radius:0)、偏移阴影(2px 2px 0)、衬线字体(Noto Serif SC)
2. **统一样式方案**：CSS变量 + Tailwind类，逐步淘汰inline style
3. **组件库激活**：页面通过组件prop控制变体，不再直接写CSS类
4. **Masthead精简**：去掉ornaments装饰文字，保留title+subtitle+日期
5. **目标设备收口**：当前正式设计只服务 `WDB` 移动应用；桌面浏览器只承担调试与演示预览，不再反向决定 UI 方案

## 3. 当前边界

### 3.1 不在本主线处理的内容

1. 不改动后端API、数据库schema、接口契约
2. 不新增页面功能或业务逻辑
3. 不改动路由结构或懒加载策略
4. 不引入新的设计系统（如shadcn/ui、MUI等）
5. 暗色模式、响应式断点、错误边界归入批次7（架构层），不在前6批次处理

### 3.2 必须保持的原则

1. `npm.cmd run check` 全绿是唯一准入门槛
2. 每批次修改后必须复验
3. 不引入新的依赖（除非必要且已评估）
4. 视觉风格保持"复古报纸"定位，不做风格切换
