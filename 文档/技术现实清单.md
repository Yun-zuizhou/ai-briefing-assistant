## 1. 正式源码面

## 2. 唯一运行入口

- Windows 标准入口：
  - `npm.cmd run setup`
  - `npm.cmd run dev`
  - `npm.cmd run check`
- 为减少重复试错，当前高频一键命令统一收口到 `task:*` 命名空间：
  - `npm.cmd run task:init`
  - `npm.cmd run task:start`
  - `npm.cmd run task:start:web`
  - `npm.cmd run task:verify`
  - `npm.cmd run task:access:d1`
  - `npm.cmd run task:access:workers`
- 上述 `task:*` 是稳定快捷入口，不替代项目级基础口径 `setup / dev / check`
- `sitecustomize.py` 会在仓库根自动补入 `.pydeps_runtime` 与 `.pydeps`，确保本地 `python` 与 npm 包装入口使用同一套依赖解析口径
- `tools/scripts/_shared.mjs` 继续作为 Node 脚本统一 Python 环境的唯一入口
- `vite` 当前已开启 `strictPort`
- `npm.cmd run setup` 当前会自动补：
  - 根目录 `.env`
  - `apps/web/.env`
  - `apps/edge-worker/.dev.vars`

## 3. 唯一准入门槛

- `npm.cmd run check` 是唯一准入门槛
- 当前测试数字与最近一次复验结果，以 `文档/进行中/当前测试与验收总表.md` 为准
- 当前门槛包含：
  - `apps/web` build
  - `apps/web` lint
  - Python 测试主套件
  - Workers `typecheck + vitest`
- `npm.cmd run test:api-mainline` 仅保留为辅助诊断
- `npm.cmd run preflight:workers-config` 当前可用于补充核验：
  - 本地 `apps/edge-worker/.dev.vars`
  - 默认 Worker 的 `INTERNAL_API_TOKEN` secret
- `npx.cmd playwright test e2e/app-mainline.spec.ts` 当前可用于补充核验：
  - 正式鉴权路由拦截
  - 登录后 session 恢复
  - 主线页面浏览器级交互

## 5. mock / 虚拟 / 正式边界

- `apps/web/demo/mock-data` 仅用于演示、导出模板和本地 bootstrap 种子
