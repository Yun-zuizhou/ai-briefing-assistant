本研究设计并实现了一款以"每日简报"为核心载体的个人信息管理应用——简报助手App。针对当前信息过载、信息整理困难、信息感知不足等痛点，本系统通过智能提炼将分散的信息整合为高度凝练的简报，主动帮助用户从信息海洋中获取有价值的内容。系统前端采用React + TypeScript + Tailwind CSS技术栈，后端经历了从Python + FastAPI到TypeScript + Hono + Cloudflare Workers的演进，数据库采用Cloudflare D1（SQLite兼容），并支持DeepSeek、OpenAI、Anthropic、Gemini等7家大模型提供商接入。系统实现了基于规则与关键词匹配的意图识别、兴趣驱动的内容推荐排序、会话持久化与多轮对话、周期报告生成等核心功能。本文重点阐述了AI工具在技术选型、数据库设计、接口开发及大模型接入中的具体应用，探索了"简报驱动"的信息交互模式，为解决信息过载问题提供了新的思路和方法。

### 外文摘要（Abstract）

（250—300个词）

This paper designs and implements a personal information management application called "Briefing Assistant App" with "Daily Briefing" as its core carrier. Aiming at the pain points of current information overload, difficulty in information organization, and insufficient information awareness, the system proactively helps users obtain valuable content from the ocean of information by intelligently refining scattered information into highly condensed briefings. The frontend is implemented using React + TypeScript + Tailwind CSS. The backend evolved from Python + FastAPI to TypeScript + Hono + Cloudflare Workers. The database uses Cloudflare D1 (SQLite-compatible), and the system supports integration with 7 major AI providers including DeepSeek, OpenAI, Anthropic, and Gemini. Core features include rule-based intent recognition with keyword matching, interest-driven content recommendation, session persistence with multi-turn dialogue, and periodic report generation. This paper focuses on the specific application of AI tools in technology selection, database design, API development, and large model integration, exploring a "briefing-driven" information interaction model.

**Key words:** Personal Information Management; Intelligent Briefing; Intent Recognition; Large Language Model; AI-Assisted Development; Cloudflare Workers; React

- 1.1.1 信息过载时代的个人信息管理困境

  - 用户缺乏对主题发展趋势的感知能力

## 论文格式说明与注意事项

### 图表编号规则

- 图表不多的话：图1—图20, 表1—表20（一贯到底）
- 图表多的话：分章编号，如：图2-1（图2.1），表2-1（表2.1）
- 图题放在图的正下方，表题放在表的正上方
- 图表及图题、表题的文字比正文减小一个字号
- 建议用三线表，表格转页应重复表头并在右上角标"（续表）"

### 公式编号规则

- 一律采用阿拉伯数字分章依序编排
- 如："式（2-13）" "式（4-5）"
- 标注应于该公式所在行的最右侧
- 公式书写方式应在文中相应位置另起一行居中横排
- 较长的公式只可在符号处（+、-、\*、/、≤≥等）转行

### 章节编号格式

- 章："1" "2" "3" ...
- 节："1.1" "1.2" ...
- 小节："1.1.1" "1.1.2" ...
- 或采用："一、" "（一）" "1." "（1）" 等格式

### 其他注意事项

- 正文用中文撰写，字数要求一般为8000字
- 摘要采用第三人称撰写，用语规范，明确简练
- 关键词间用分号隔开，列出3～6个
- 中外文标点不混用
- 必要的地方应随页加脚注
- 正文中不应加入程序的源代码（除非必需）
- 采用国家正式公布实施的简化汉字和法定的计量单位
- 术语、符号、代号必须统一，符合规范化要求
- 新的专业术语、缩略语应加以注释
- 国外新的专业术语、缩略语，必须在译文后用小括号注明原文
