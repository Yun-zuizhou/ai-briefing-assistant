本文设计并实现了一款以"每日简报"为核心载体的个人信息管理应用——简报助手App。针对当前信息过载、信息整理困难、信息感知不足等痛点，本系统通过智能提炼将分散的信息整合为高度凝练的简报，主动帮助用户从信息海洋中获取有价值的内容。系统采用React + TypeScript + Tailwind CSS技术栈实现前端界面，采用Python + FastAPI框架构建后端服务，集成DeepSeek API实现意图识别、简报生成与信息提炼等AI功能，使用SQLAlchemy ORM + SQLite完成数据存储与管理。本文重点阐述了AI工具在技术选型、数据库设计、接口开发及大模型接入中的具体应用，探索了"简报驱动"的信息交互模式，为解决信息过载问题提供了新的思路和方法。

**关键词：** 个人信息管理；智能简报；意图识别；大语言模型；AI辅助开发；FastAPI；React

### 外文摘要（Abstract）

（250—300个词）

This paper designs and implements a personal information management application called "Briefing Assistant App" with "Daily Briefing" as its core carrier. Aiming at the pain points of current information overload, difficulty in information organization, and insufficient information awareness, the system proactively helps users obtain valuable content from the ocean of information by intelligently refining scattered information into highly condensed briefings. The frontend is implemented using React + TypeScript + Tailwind CSS, while the backend is built with Python + FastAPI framework. The system integrates DeepSeek API to implement AI functions such as intent recognition, briefing generation, and information refinement, and uses SQLAlchemy ORM + SQLite for data storage and management. This paper focuses on the specific application of AI tools in technology selection, database design, API development, and large model integration, exploring a "briefing-driven" information interaction model.

**Key words:** Personal Information Management; Intelligent Briefing; Intent Recognition; Large Language Model; AI-Assisted Development; FastAPI; React

### 第1章 绪论

#### 1.1 研究背景与意义

- 1.1.1 信息过载时代的个人信息管理困境
  - 信息分散在多个平台，用户在各平台间来回跳转
  - 信息获取后缺乏有效记录和沉淀机制
  - 用户缺乏对主题发展趋势的感知能力
- 1.1.2 人工智能技术在个人信息管理领域的应用前景
  - 大语言模型在文本处理、意图理解方面的能力
  - AI技能从桌面端向移动端迁移的趋势
- 1.1.3 研究目的与意义
  - 理论意义：探索"简报驱动"的信息交互模式，构建动态演化用户画像
  - 实践意义：降低信息管理学习成本，提升信息获取效率
  - 社会意义：为解决信息过载问题提供新思路

#### 1.2 国内外研究现状

- 1.2.1 信息聚合类应用发展现状
  - 语鲸、即刻、小红书等产品的功能特点与不足
  - Notion AI、Readwise等工具的学习成本与本地化问题
- 1.2.2 AI辅助开发工具发展现状
  - AI辅助代码生成、设计、测试的工具演进
  - Trae IDE等AI辅助开发平台的出现
- 1.2.3 用户画像与智能摘要技术研究现状
  - 静态画像与动态演化画像的研究进展
  - 多源信息整合与智能摘要技术
- 1.2.4 现有研究的不足与本课题的切入点

- 图表不多的话：图1—图20, 表1—表20（一贯到底）

- 图题放在图的正下方，表题放在表的正上方

- 建议用三线表，表格转页应重复表头并在右上角标"（续表）"

### 公式编号规则

- 一律采用阿拉伯数字分章依序编排
- 如："式（2-13）" "式（4-5）"

- 公式书写方式应在文中相应位置另起一行居中横排
- 较长的公式只可在符号处（+、-、\*、/、≤≥等）转行
