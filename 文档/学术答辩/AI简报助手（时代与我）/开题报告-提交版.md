## 一、选题背景、目的及意义

### 1．研究背景

信息已成为个人成长和职业发展的重要资源。随着信息技术的快速发展，互联网已成为人们获取信息的主要渠道。然而，当前的信息获取与管理模式存在诸多问题。第一，信息过载问题：信息分散在多个平台，想要对新资讯获得足够的了解，就要求用户在各平台之间来回跳转，这严重影响了信息获取效率。第二，信息整理问题：用户获取信息后缺乏有效的记录和沉淀机制，大量有价值的信息在浏览后即被遗忘，零散的记录难以整理，多个应用间的输入难以整合，专门学习一个记录工具又要花费巨大的学习成本和习惯养成成本，难以快速转化为有效的个人知识资产。第三，信息感知不足：用户往往停留在"知道"层面，缺乏对某主题发展趋势的感知，甚至无法看到自己已知信息的脉络和轨迹，缺乏持续学习和进步的动力反馈。

与此同时，人工智能技术发展迅速，大语言模型在文本处理、意图理解等方面展现出强大能力，为个人信息管理提供了新的技术手段。特别是AI技能日趋成熟后，桌面端用户已能够通过AI配置自己需要的信息摘要、每日更新。移动端虽不像桌面端一样方便操控代码和文件，但可以将这一套获取信息的方式沉淀为应用程序，让更多用户受益。通过调研发现，当前市场上的信息聚合类应用存在明显不足：语鲸等产品专注于信息摘要，但缺乏个性化定制和用户画像能力；各大社交媒体平台（如微博、抖音、小红书等）虽然具备智能推荐能力，但基于兴趣算法的推荐机制容易让用户陷入信息茧房；Notion AI等功能强大但学习成本高；Readwise等国外产品缺乏中文支持。因此，本课题提出"简报助手App"——一款以"每日简报"为核心载体的个人信息管理应用，通过智能提炼将分散的信息整合为高度凝练的简报，主动帮助用户从信息海洋中获取有价值的内容，呈现信息的变化脉络，记录思考轨迹，见证个人成长。

### 2．研究目的

本课题旨在设计并开发一款以"每日简报"为核心载体的个人信息管理应用，实现以下目标：

第一，智能提炼，AI主动总结、加工信息，生成高度凝练的简报，而非简单推送原文；

第二，趋势洞察，呈现信息的变化脉络，提供周报、月报、季报等多维度趋势分析，帮助用户把握发展趋势；

第三，意图识别，通过自然语言交互降低用户配置门槛，用户可直接提出需求，无需学习复杂的功能操作；

第四，信息整理，整合用户零碎的想法和观点，转写成逻辑性、文学性强的叙述，提供文字总结和及时反馈，缓解信息焦虑，清晰记录用户思考轨迹；

第五，用户画像，从信息关注、行动力等角度分析生成用户画像，帮助用户更了解时代，也更了解自己；

### 3．研究意义

在理论意义上，本课题将用户画像、智能摘要、意图识别等技术应用于个人信息管理领域，探索"简报驱动"的信息交互模式。用户画像是通过分析用户数据，构建用户特征模型的，用于理解用户需求和行为的一种方法，现有研究以静态画像为主。本课题尝试构建动态演化画像，通过持续追踪用户行为，生成成长轨迹，并以故事化方式呈现。智能摘要技术是自然语言处理的重要应用领域，本课题尝试探索多源信息整合与提炼的方法，将分散在不同平台的信息整合为高度凝练的简报。意图识别是对话系统的核心技术，本课题尝试将大语言模型应用于移动端个人信息管理场景，探索自然语言交互降低用户学习成本的可行性。

在实践意义上，简报助手App能够帮助用户从信息海洋中快速获取有价值的内容，通过智能提炼呈现信息的变化脉络，记录思考轨迹，见证个人成长，降低信息管理的学习成本，提升使用效率。对于学术研究者，可自动推送相关领域文献；对于求职者，可集中推送兼职信息并智能提醒截止日期；对于行业关注者，可一站式聚合行业资讯；对于自我成长者，可AI辅助整理想法并可视化成长轨迹。通过意图识别支持自然语言交互，降低用户的学习成本。针对自我认知问题，尝试从信息关注、行动力等角度生成用户画像，帮助用户了解自己的信息行为特征。

在社会意义上，本课题探索了人工智能在个人信息管理领域的垂直应用，为解决信息过载问题提供了新的思路和方法，随着AI技能在桌面端的成熟应用，本课题尝试将其构建思路迁移到移动端程序，让更多用户能够接触到AI辅助信息管理的便利，具有一定的实践参考价值。

## 二、研究内容和研究方法

### 1．研究内容

#### （一）初步设计

本课题拟设计一款以"每日简报"为核心载体的个人信息管理应用。简报是产品的核心载体，整合了信息追踪、待办提醒、个人叙事三大内容模块。用户通过对话式交互完成所有配置和操作，系统通过智能提炼生成高度凝练的简报，并支持用户回复简报完成记录想法、创建待办等操作。

本课题的创新点在于：第一，智能提炼，AI主动总结、加工信息，生成高度凝练的简报；第二，趋势洞察，呈现信息的变化脉络；第三，多版本输出，提供不同字数版本的简报；第四，互动闭环，回复简报即可记录、创建待办。

#### （二）主要工作内容

**1) 前期准备**

a）查阅用户建模、个性化推荐、对话式交互等相关资料，了解目前AI技术在个人信息管理领域的应用现状以及发展趋势。

b）调研市场上现有的信息聚合类应用，如即刻、小红书、Notion AI、Readwise等，对比分析主要功能特点、用户痛点、交互方式等。

c）选取目标用户群体作为调研对象，设计调研问卷，调查用户在信息获取与管理方面的痛点、需求以及期望获得的AI帮助类型。

d）整理调研结果和访谈内容，确定App的核心功能定位。

**2) App设计**

a）简报生成与智能提炼功能设计：设计信息收集与筛选机制，实现多源信息的聚合；设计智能摘要与提炼算法，将分散的信息整合为高度凝练的简报；设计趋势分析与变化脉络呈现功能；设计多版本输出策略，提供不同字数版本的简报。

b）用户画像功能设计：设计用户行为追踪机制，从信息关注、行动力等维度构建用户自画像；设计基于行为数据的多维建模方案，生成文字性的用户画像描述和呈现。

c）对话式交互与意图识别功能设计：设计意图识别算法，支持用户通过对话完成配置、记录、待办等操作；设计多轮对话管理机制；针对简报互动场景进行专门优化。

d）界面设计：使用 Trae-IDE 平台（国内版）进行界面原型设计，通过自然语言描述设计需求，调用前端设计 skills 生成图标和概念设计，采用"提出需求-生成-反馈-修改"的迭代方式逐步优化界面。

**3) App原型迭代**

a）前端开发：采用React + TypeScript + Tailwind CSS技术栈，完成前端界面设计与实现。

b）后端开发：采用Python + FastAPI框架，完成后端服务架构设计、API接口开发。

c）AI模块集成：集成DeepSeek API，实现意图识别、简报生成、信息提炼等功能；后续可根据需要扩展支持其他大语言模型接口。

d）数据库设计：采用SQLAlchemy ORM框架，使用SQLite作为数据库（MVP阶段），设计用户、意图、内容、机会、计划、画像等数据表，完成数据存储与管理。

**4）用户测试**

在功能基本完成后，邀请目标用户进行测试。测试App的各项功能，收集用户的反馈和改进建议。重点关注简报内容质量和交互易用性，根据反馈调整和优化系统。

**本毕设所需软件**：

- 设计工具：AI辅助设计工具（Trae IDE），通过自然语言交互完成界面设计
- 前端开发：React、TypeScript、Tailwind CSS、Vite
- 后端开发：Python、FastAPI、SQLAlchemy
- AI模块：DeepSeek API
- 数据库：SQLite
- 开发环境：Trae IDE
- 版本控制：Git

### 2．研究方法

（1）文献研究：查阅用户建模、智能摘要、意图识别等相关文献和资料，了解现有技术和应用案例。

（2）竞品分析：调研语鲸、即刻、小红书、Notion AI、Readwise等信息聚合类应用，分析其功能特点和用户痛点。

（3）用户调研：通过问卷和访谈收集目标用户在信息获取与管理方面的需求和痛点。

（4）原型设计：使用AI辅助工具进行界面原型设计，通过迭代优化完善设计方案。
