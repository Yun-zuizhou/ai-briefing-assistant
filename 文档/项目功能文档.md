## 1. 这个项目是干什么的

- **项目名称**：AI简报助手（时代与我）
- **这个项目一句话是做什么的**：每天一份简报，帮你收集世界，记录自己，看见成长
- **主要给谁用**：需要高效获取信息、管理个人关注领域、追踪热点趋势的用户
- **这个项目想解决什么问题**：信息过载时代，帮助用户从海量信息中筛选有价值的内容，通过AI对话简化操作，形成个人知识沉淀
- **这一版最核心的目标是什么**：在核心正式读侧已基本站稳后，继续完成周期报告详情页与行动页尾差的前端产品化总收口

## 3. 用户会怎么使用这个项目

**用户第一次进入会看到什么：**
- 欢迎引导页，展示3个核心功能介绍（滑动切换）
- 最后一张点击"开始使用"进入登录页

**用户接下来通常会点什么：**
- 登录/注册账号
- 设置感兴趣的领域（首次登录引导）
- 查看首页的每日简报
- 点击"对话"尝试AI交互

**用户完成最核心任务的大概步骤是什么：**
1. 打开App → 欢迎引导 → 登录/注册
2. 设置关注领域 → 查看个性化简报
3. 点击"对话" → 输入"我想关注AI" → AI自动添加关注
4. 每天收到推送简报 → 查看热点 → 收藏感兴趣的内容

**如果这是一个有登录的系统，登录前后分别会发生什么：**
- **登录前**：只能看到欢迎引导页和登录页
- **登录后**：
  - 首次登录进入兴趣配置页
  - 之后进入首页查看简报
  - 可以使用对话、待办、收藏等所有功能
  - 数据会保存到个人账户

## 5. 目前还没想清楚的地方

**当前仍在收束、不能误判为已完成的点：**
1. **统一内容层仍未进一步实体化**：`content_ref + /api/v1/content/by-ref` 已稳定，但尚未进入更完整统一事实层。
2. **推荐与加工仍是最小规则实现**：已有最小加工字段，但还没有正式排序体系、可信度体系与统一推荐权重。
3. **报告页当前已完成第一轮收口**：周报页已去掉过渡态文案与 alert 分享，月报 / 年报保持最小分享导出能力。
4. **行动页当前已完成第一轮收口**：连续打卡已改走正式 `growth-overview`。
5. **记录类意图识别仍有抢占风险**：部分自然语言记录表达仍可能被 `create_todo` 规则提前吸走。

**哪些地方希望 AI 先给建议：**
1. 统一内容层后续实体化策略：何时从稳定读侧进入正式 `contents` 或等价统一事实层
2. D1 切换验收设计：第一批行为事实接口云端主读后，如何补最小专项回归
3. Chat 分流和纠偏优化：如何降低记录类输入被待办规则抢占
4. 内容加工升级：如何从当前规则型过渡实现走向正式排序与推荐体系
