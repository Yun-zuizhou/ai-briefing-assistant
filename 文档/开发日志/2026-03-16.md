## 🔑 关键决策
1. **决策**: `.app` 使用 `calc(100% - 4px)` → **原因**: `#root` 有 2px border，上下共 4px
2. **决策**: 导航栏从凸起式改为平铺式 → **原因**: 凸起按钮会被 overflow 裁剪，平铺更稳定
3. **决策**: 添加金色装饰线 → **原因**: 符合复古报纸风格设计系统
4. **决策**: 使用SHA256+salt替代bcrypt → **原因**: bcrypt有72字节密码长度限制，SHA256更灵活
5. **决策**: 降低质量评分阈值至0.3 → **原因**: 部分文章摘要较短导致评分为0

## ⚠️ 遇到的问题
- **导航栏完全在容器外面**: 初始以为是 overflow 问题 → 实际是容器高度计算问题
- **中间凸起按钮被裁剪**: `overflow: hidden` 会裁剪超出部分 → 改为居中平铺设计
- **底部白色边边**: `.app` 没有考虑顶部 border → 添加 `margin-top: 2px`
- **机器之心RSS返回0条**: 需要内测申请 → 移除该RSS源
- **AI科技大本营404**: RSS源不存在 → 移除该RSS源
- **文章质量分数为0**: 摘要长度要求过高 → 调整评分逻辑
- **datetime序列化错误**: Pydantic模型datetime对象无法JSON序列化 → 显式转为ISO字符串
- **bcrypt密码长度限制**: 超过72字节密码报错 → 改用SHA256+salt

## 💭 今日反思
- CSS 布局问题要从外层容器逐层排查
- `overflow: hidden` 会裁剪所有超出内容，包括设计需要的凸起效果
- 边框（border）会占用空间，需要在高度计算中考虑
- 复古报纸风格的导航栏应该简洁、有质感，不需要过度装饰
- RSS源需要定期检查可用性，失效源应及时移除
- bcrypt有72字节限制，长密码场景应使用其他哈希算法
- datetime序列化问题在FastAPI中常见，需显式转换
