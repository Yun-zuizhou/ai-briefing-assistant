## 1. 3 月 17 日之后具体推进了哪些工作

### 阶段一：真实接口过渡态建立与工程收口（3 月底 - 4 月 3 日）

这一阶段的核心推进，是把“原型 / 页面草稿”开始切到真实接口和正式工程链路上。

具体推进包括：

1. 主要页面逐步切到真实接口过渡态。
2. 打通 `Chat -> interest -> Today` 等关键交互基础链路。
3. 恢复前端 `build / lint` 稳定通过，收掉早期工程噪音。
4. 收口默认 `pytest` 入口，使测试从零散运行转向主项目统一入口。

阶段结果是：项目从“能展示的原型”开始转向“可联调、可验证的正式工程形态”。

### 阶段二：Today / 统一内容读侧 / D1 切换验证（4 月 3 日 - 4 月 4 日）

这一阶段重点从页面接线，进一步推进到内容读侧和正式数据库切换。

具体推进包括：

1. 收束 `Today / content_ref / by-ref / ArticlePage` 统一内容读侧。
2. 让 `dashboard/today`、`reports/*`、`content/by-ref` 完成 D1 主读切换。
3. 把阶段主线从“页面接真实接口”升级为“Cloudflare D1 正式数据库切换验证”。
4. 明确统一内容层、行为事实层、Chat 分流表达等边界，不再无序扩面。

阶段结果是：正式数据源从本地过渡态走向 D1 主读，Today 主链开始具备真实数据基础。

### 阶段三到阶段五：内容加工层、Chat 会话层、报告对象层正式化（4 月 4 日 - 4 月 5 日）

这一轮属于“对象层正式化”推进。

具体推进包括：

1. 完成 `article_processing_results` 与 `hot_topic_processing_results` 的最小正式落位。
2. 对 `contents` 统一事实总表做出“继续后置、不立即实现”的结果级决策。
3. 建立 `chat_session / chat_message` 会话消息持久化最小实现，并完成远端 apply。
4. 建立 `reports` 正式承接位，形成周报 / 月报的正式闭环。
5. 明确 `annual` 先后置，不在当下立即扩实现。

阶段结果是：内容加工、Chat 会话、报告快照这三类“正式对象”开始站稳，项目不再只是页面拼装。

### 阶段十到阶段十四：页面正式读侧整合与产品化收口（4 月 5 日 - 4 月 7 日）

这一轮的重点，已经不再是“补更多对象”，而是把已有正式对象真正拉回到产品页面里。

具体推进包括：

1. `TodayPage` 基本站到 `dashboard/today` 正式读侧。
2. `GrowthPage` 切到 `growth-overview`。
3. `ActionsPage` 切到 `actions/overview`，并补齐 `today / future / completed` 正式分组。
4. `HistoryBriefPage` 开始真正消费 `getReports()`，形成“周期回顾 -> 历史回看”闭环。
5. `/me` 切回独立 `MyPage`，形成“我的页 / 个人沉淀入口”。
6. 收 `WeeklyReportPage` 产品化表达，去掉明显“过渡态”文案和 alert 级分享。
7. 去掉 `ActionsPage` 的 mock streak 依赖，改用正式读侧数据。
8. 做一轮云端主线优化，压缩 Today / Growth / Actions 的函数级耗时。

阶段结果是：主闭环页面基本不再依赖手工拼旧接口，前端产品面开始真正稳定。

### 阶段十五：Cloudflare Workers 迁移与云端上线（4 月 13 日）

这一轮是本周期最重要的技术迁移节点。

具体推进包括：

1. 放弃 Railway 方案，统一采用 `Cloudflare Workers + Pages + D1`。
2. 完成后端从 `Python FastAPI` 到 `TypeScript + Hono` 的正式在线主线迁移。
3. 分阶段迁移核心 API，最终形成 23 个正式端点。
4. 前端切到 Cloudflare Pages，后端切到 Workers，形成正式云端部署。
5. 保留 Python `app/` 作为离线 / 参考 / 验证链，而不再作为正式在线主后端。

阶段结果是：项目完成“正式在线后端”的主线切换。

### 当前阶段：完整链路、支撑链、仓库收口与正式化标准（4 月 16 日）

这是当前最新一轮推进，目标已经从“主链能跑”继续升级为“完整链路可验收、技术边界可长期维护”。

## 3. 哪些技术点出现了技术迁移和调整

本轮最明显的技术迁移和调整主要集中在 6 个方面：

### 3.1 后端主线迁移

1. 正式在线后端由 `Python FastAPI` 迁移为 `TypeScript + Hono + Cloudflare Workers`。
2. Python `app/` 从“主后端”调整为“离线 / 参考 / 验证链”。

### 3.2 数据源迁移

1. 正式数据主线由本地 `SQLite` 调整为云端 `Cloudflare D1`。
2. 本地 SQLite 仍保留，但角色已调整为开发过渡态和本地验证态。

### 3.3 页面读侧架构调整

1. 从页面手工拼旧接口、局部依赖 mock 数据，调整为聚合读侧主导。
2. 典型表现是：
   - `TodayPage -> dashboard/today`
   - `ActionsPage -> actions/overview`
   - `GrowthPage -> growth-overview`

### 3.4 内容体系调整

1. 从多入口、多对象直接跳详情，调整为 `content_ref + content/by-ref` 统一内容引用。
2. 内容加工结果表独立落位，统一内容总表 `contents` 被明确后置，不盲目扩实现。

### 3.5 契约与仓库结构调整

1. 从前后端各自维护类型，调整为 `packages/contracts/src/page-data.ts` 单一源。
2. 从 `prototype / workers / scripts / cloudflare/d1` 的旧目录口径，迁移为：
   - `apps/web`
   - `apps/edge-worker`
   - `tools/scripts`
   - `infra/cloudflare/d1`
   - `var/local`
   - `var/logs`

### 3.6 验证口径调整

1. 从零散脚本和局部命令，调整为统一门槛体系。
2. 目前已经形成：
   - 主门槛：`npm.cmd run check`
   - Workers 专项门槛：`npm.cmd run check:workers`
   - 阶段验收口径：`npm.cmd run check:stage`
   - 云端同步预检：`npm.cmd run preflight:workers-sync`

## 4. 当前最新进度和优化方向

### 4.1 当前最新进度

截至今天，项目已经进入“Cloudflare Workers 迁移完成后”的第二阶段，即：

`产品层尾差清理 + 完整链路收口 + 正式化标准建立`

当前最新状态可以概括为：

1. 主链路已经跑通，不再停留在原型或接口过渡态。
2. 次链路与支撑链正在补“最小事实层”，项目从“主链可跑”升级为“完整链路可追溯”。
3. 仓库结构与接口契约已经大幅收口，协作成本明显下降。
4. 当前文档和代码的重点，已经不是“做更多功能”，而是“把正式链边界定死，避免回退”。

### 4.2 当前最重要的优化方向

结合今天的进行中文档，后续优化方向已经比较明确：

1. **鉴权与权限链**
   - 去掉 `user_id query`、`x-user-id`、`x-user-email` 这类伪身份入口
   - 建立真正的认证上下文和正式鉴权链

2. **Chat 正式写链收口**
   - 继续清退前端失败时的本地“假成功写回”
   - 把 Chat 严格收成“后端成功才算正式成功”

3. **本地状态边界清理**
   - 清掉 `localStorage` 对正式身份和正式业务事实的真源角色
   - 本地状态只保留 UI cache / draft / 体验兜底

4. **报告可信度修正**
   - 对 `insufficient_data`、关键指标定义、趋势值来源做严格收口
   - 避免“看起来真实但不可追溯”的报告结果继续混入正式态

5. **支撑链执行化**
   - 当前支撑链已具备事实层与观测层
   - 下一步要把 `/api/v1/system/*` 从“可观测接口”继续推进到“真实调度 / 加工执行链”

6. **人工验收与截图证据补齐**
   - 重点补 `Today / Actions / Growth / My / Weekly`
   - 尤其需要补“关注 AI -> 推荐命中 AI”等主链场景的截图级验收

7. **Python app 安全迁移准备**
   - 当前不能硬迁 `app -> apps/api`
   - 下一步应先做兼容层、路径抽象和入口可切换，再决定是否进入物理迁移

### 4.3 对当前阶段的总体判断

如果用一句话概括当前进度：

`项目已经完成“从原型 / FastAPI 主线 / 本地过渡态”向“Workers + Pages + D1 正式主线”的关键跃迁，当前正在从“主链能跑”继续收口到“正式链可信、完整链可验收、仓库边界可长期维护”。`
