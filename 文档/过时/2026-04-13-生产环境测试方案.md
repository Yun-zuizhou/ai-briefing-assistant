## 1. 测试方针

### 1.1 问题分类

| 级别 | 定义 | 处理方式 |
|------|------|---------|
| **P0 阻断** | 核心功能无法使用，用户无法完成基本任务 | 立即修复 |
| **P1 严重** | 重要功能异常，影响用户体验 | 记录，优先修复 |
| **P2 一般** | 功能可用但有瑕疵，不影响核心流程 | 记录，按优先级修复 |
| **P3 轻微** | 小问题，不影响使用 | 顺手修复或记录 |

### 1.2 "顺手修复"标准

满足以下条件的问题可以顺手修复：
- 修复时间 < 5 分钟
- 不需要修改核心逻辑
- 不引入新风险
- 不影响其他测试进行

### 1.3 问题记录格式

```
### 问题 N：[问题标题]
- **级别**：P0/P1/P2/P3
- **页面**：页面名称
- **复现步骤**：
  1. 步骤一
  2. 步骤二
- **预期结果**：xxx
- **实际结果**：xxx
- **状态**：待修复 / 已修复 / 需讨论
```

## 3. 测试结果汇总

### 3.1 问题列表

### 问题 1：CORS 配置错误（已修复）
- **级别**：P0
- **页面**：全局
- **复现步骤**：
  1. 访问 https://ai-briefing-5d0.pages.dev
  2. 前端尝试调用 API
- **预期结果**：API 请求成功
- **实际结果**：CORS 错误，所有 API 请求被阻止
- **状态**：✅ 已修复

**原因**：后端 CORS 配置中的 origin 列表不包含实际的前端 URL `https://ai-briefing-5d0.pages.dev`

**修复**：在 `workers/src/index.ts` 中添加正确的 origin

### 问题 2：意图识别不准确（已修复）
- **级别**：P2
- **页面**：Chat
- **复现步骤**：
  1. 调用 `/api/v1/chat/recognize`
  2. 发送 `{input: "关注AI写作"}`
- **预期结果**：识别为 `add_interest` 意图
- **实际结果**：识别为 `chat_only` 意图（confidence: 0.5）
- **状态**：✅ 已修复（2026-04-15）

**原因分析**：
1. 后端 `parseIntent` 函数的 `add_interest` 检测逻辑过于简单
2. 只检查 `['关注', '订阅', '追踪']` 三个关键词
3. 需要同时满足：文本包含 `currentInterests` 中的某个兴趣 AND 包含关注关键词
4. 缺少兴趣关键词映射表，无法从文本中提取兴趣

**修复内容**：
1. 添加 `interestKeywords` 映射表（10个兴趣领域，50+关键词）
2. 添加 `addInterestPatterns` 和 `removeInterestPatterns` 正则数组
3. 添加 `negativePrefixes` 负向前缀检查
4. 添加 `extractInterests` 函数从文本中提取兴趣
5. 重写 `parseIntent` 函数，优先检测兴趣相关意图

**修复文件**：`workers/src/routes/chat.ts`

**验证方法**：
```bash
curl -X POST https://ai-briefing-assistant.aibriefing2026.workers.dev/api/v1/chat/recognize \
  -H "Content-Type: application/json" \
  -d '{"input": "关注AI写作"}'
```

**预期返回**：
```json
{
  "recognized_intent": "add_interest",
  "confidence": 0.85,
  "extracted_entities": { "interests": ["AI"] }
}
```
