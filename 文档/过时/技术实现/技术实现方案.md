## 一、对话式首页设计

### 1.1 设计方案对比

| 维度 | 对话式首页 | 传统卡片式首页 |
|------|-----------|---------------|
| **交互方式** | 自然语言输入 | 点击、滑动 |
| **学习成本** | 低（人人都会聊天） | 中（需要理解界面） |
| **信息密度** | 低（一屏一对话） | 高（多卡片展示） |
| **个性化程度** | 极高（AI响应） | 中（预设推荐） |
| **开发复杂度** | 高（需要LLM集成） | 中（常规UI） |
| **响应速度** | 慢（生成需要时间） | 快（即时展示） |

#### 对话式首页的优势

| 优势 | 说明 |
|------|------|
| **契合定位** | "时代与我"强调个人与时代的对话，聊天框天然是"对话"的隐喻 |
| **降低门槛** | 用户不需要思考"怎么用"，直接说"我想看AI相关的机会" |
| **引导观点输出** | 聊天场景下，用户更容易表达观点："我觉得这个热点挺有意思的" |
| **统一入口** | 所有功能通过对话触达，无需复杂导航 |
| **冷启动友好** | 新用户可以直接问"你能帮我做什么？" |

#### 对话式首页的挑战

| 挑战 | 说明 |
|------|------|
| **信息浏览效率低** | 用户想快速浏览多个热点，聊天框需要逐个询问 |
| **LLM成本** | 每次对话都调用API，成本较高 |
| **响应延迟** | 生成回复需要时间，可能影响体验 |
| **功能发现性差** | 用户不知道APP能做什么，需要主动询问或引导 |
| **毕设技术难度** | 需要集成LLM，增加开发复杂度 |

### 1.4 技术实现方案

#### 方案对比

| 方案 | 复杂度 | 说明 |
|------|--------|------|
| **方案A：纯对话式** | 高 | 全部交互通过LLM，成本高、延迟大 |
| **方案B：混合式** | 中 | 对话+卡片，LLM仅用于生成回复 |
| **方案C：伪对话式** | 低 | 预设对话模板，点击触发，不调用LLM |

**推荐方案B：混合式**

#### 数据模型

```python
# 对话消息结构
class ChatMessage(Base):
    __tablename__ = "chat_messages"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    
    role = Column(String(20))      # "user" / "assistant"
    content = Column(Text)         # 文本内容
    
    # 嵌入式卡片（可选）
    card_type = Column(String(50), nullable=True)  # "hot_topic" / "opportunity" / "profile"
    card_data = Column(Text, nullable=True)        # JSON格式的卡片数据
    
    created_at = Column(DateTime, default=datetime.utcnow)
```

#### 前端组件

```typescript
// 对话消息组件
const ChatMessage = ({ message }: { message: ChatMessage }) => {
  if (message.card_type) {
    return <EmbeddedCard type={message.card_type} data={message.card_data} />;
  }
  
  return (
    <View style={message.role === 'user' ? styles.userBubble : styles.aiBubble}>
      <Text>{message.content}</Text>
    </View>
  );
};

// 嵌入式卡片组件
const EmbeddedCard = ({ type, data }: { type: string; data: any }) => {
  switch (type) {
    case 'hot_topic':
      return <HotTopicCard topic={data} />;
    case 'opportunity':
      return <OpportunityCard opportunity={data} />;
    case 'profile':
      return <ProfileCard profile={data} />;
    default:
      return null;
  }
};
```

#### 后端API

```python
# 对话接口
@router.post("/chat")
async def chat(
    message: str,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user)
):
    """处理用户对话"""
    
    # 1. 保存用户消息
    user_message = ChatMessage(
        user_id=current_user.id,
        role="user",
        content=message
    )
    db.add(user_message)
    
    # 2. 解析用户意图
    intent = parse_intent(message)
    
    # 3. 根据意图生成回复
    if intent == "ask_opportunities":
        opportunities = get_opportunities(current_user.id, db)
        response = generate_opportunities_response(opportunities)
        card_type = "opportunity"
        card_data = opportunities
    elif intent == "ask_hot_topics":
        topics = get_hot_topics(current_user.id, db)
        response = generate_topics_response(topics)
        card_type = "hot_topic"
        card_data = topics
    elif intent == "ask_profile":
        profile = get_user_profile(current_user.id, db)
        response = generate_profile_response(profile)
        card_type = "profile"
        card_data = profile
    else:
        # 使用LLM生成回复
        response = await llm_generate_response(message, current_user.id, db)
        card_type = None
        card_data = None
    
    # 4. 保存AI回复
    ai_message = ChatMessage(
        user_id=current_user.id,
        role="assistant",
        content=response,
        card_type=card_type,
        card_data=json.dumps(card_data) if card_data else None
    )
    db.add(ai_message)
    db.commit()
    
    return {
        "content": response,
        "card_type": card_type,
        "card_data": card_data
    }
```

```python
def generate_profile_text(user: User, db: Session) -> str:
    """生成用户画像（时代坐标版）"""
    
    # 1. 获取用户数据
    opinions = db.query(UserOpinion).filter(UserOpinion.user_id == user.id).all()
    feedbacks = db.query(UserFeedback).filter(UserFeedback.user_id == user.id).all()
    
    # 2. 分析兴趣领域
    interest_counter = Counter()
    for opinion in opinions:
        if opinion.topic_id:
            topic = db.query(HotTopic).get(opinion.topic_id)
            if topic and topic.categories:
                interest_counter.update(json.loads(topic.categories))
    
    top_interests = [tag for tag, _ in interest_counter.most_common(5)]
    
    # 3. 分析观点特征
    stance_analysis = analyze_stances(opinions)
    
    # 4. 生成画像
    year = datetime.now().year
    
    profile = f"在{year}年的时代图景中，你是一位"
    
    if top_interests:
        interests_text = "与".join(top_interests[:3])
        profile += f"关注「{interests_text}」议题的思考者。\n\n"
    
    # 添加观点特征
    if stance_analysis:
        profile += stance_analysis
    
    # 添加关键词
    keywords = extract_keywords(user, db)
    profile += f"\n\n你的时代关键词：{keywords}"
    
    return profile
```

### 2.5 时代计划（新增）

**定位**：追踪你的行动，陪伴你的成长

#### 核心场景

用户在聊天框输入"帮我看看兼职的资料呗"，APP帮用户配置搜索、筛选合适的兼职资料，并添加追踪关注，添加到用户的"计划"卡组中，罗列兼职的要求、招聘截止日期等信息，并智能地给出建议。每天提醒，"简历通过了吗？""开始写简历了吗？"如果用户关注了，但是一直没有投简历、面试的反馈，则提醒"放弃也是一种努力，把这张计划卡移出，看看别的机会吧"。

#### 场景价值

| 价值点 | 说明 |
|--------|------|
| **降低行动门槛** | 用户不需要自己去各个平台搜索 |
| **行动追踪** | 不只是"看"，而是"做" |
| **持续陪伴** | 每日提醒，让用户感受到关注 |
| **情感支持** | "放弃也是一种努力"这种表达很温暖 |

#### 功能描述

| 功能 | 说明 |
|------|------|
| 计划卡组 | 用户关注的机遇以卡片形式管理 |
| 状态追踪 | 待投递 → 进行中 → 已完成/已放弃 |
| 智能提醒 | 行动提醒、截止提醒、跟进提醒、放弃建议 |
| 进度统计 | 统计用户的行动完成情况 |

#### 计划卡状态流转

```
待投递 → 进行中（已投递/已报名） → 已完成/已放弃
         ↑                    ↓
         └── 跟进提醒 ←───────┘
```

#### 智能提醒设计

| 提醒类型 | 触发条件 | 提醒内容 |
|----------|----------|----------|
| **行动提醒** | 加入计划后N天未行动 | "开始写简历了吗？" |
| **截止提醒** | 距截止日期3天/1天 | "兼职申请还有3天截止" |
| **跟进提醒** | 投递后N天无回复 | "简历通过了吗？要不要跟进一下？" |
| **放弃建议** | 长期无进展 | "放弃也是一种努力，看看别的机会吧" |
| **成功庆祝** | 用户标记完成 | "太棒了！你的努力有了回报！" |

```python
from sqlalchemy import Column, Integer, String, Text, DateTime, ForeignKey
from sqlalchemy.orm import relationship
from datetime import datetime
from database import Base

class User(Base):
    __tablename__ = "users"
    
    id = Column(Integer, primary_key=True)
    username = Column(String(50), unique=True)
    email = Column(String(100), unique=True)
    hashed_password = Column(String(255))
    created_at = Column(DateTime, default=datetime.utcnow)
    
    # 关系
    profile = relationship("UserProfile", back_populates="user", uselist=False)
    opinions = relationship("UserOpinion", back_populates="user")
    feedbacks = relationship("UserFeedback", back_populates="user")
    chat_messages = relationship("ChatMessage", back_populates="user")
    plans = relationship("UserPlan", back_populates="user")


class UserProfile(Base):
    __tablename__ = "user_profiles"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), unique=True)
    interests = Column(Text, default="[]")
    behavior_stats = Column(Text, default="{}")
    profile_text = Column(Text)
    keywords = Column(Text, default="[]")
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    user = relationship("User", back_populates="profile")


class HotTopic(Base):
    __tablename__ = "hot_topics"
    
    id = Column(Integer, primary_key=True)
    title = Column(String(200))
    description = Column(Text)
    source = Column(String(100))
    categories = Column(Text, default="[]")
    guide_question = Column(String(500))
    suggested_stances = Column(Text, default="[]")
    published_at = Column(DateTime, default=datetime.utcnow)
    expires_at = Column(DateTime)


class UserOpinion(Base):
    __tablename__ = "user_opinions"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    topic_id = Column(Integer, ForeignKey("hot_topics.id"), nullable=True)
    topic_title = Column(String(200))
    content = Column(Text)
    stance = Column(String(50))
    opinion_type = Column(String(50), default="hot_topic")
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    user = relationship("User", back_populates="opinions")


class Opportunity(Base):
    __tablename__ = "opportunities"
    
    id = Column(Integer, primary_key=True)
    title = Column(String(200))
    description = Column(Text)
    type = Column(String(50))
    categories = Column(Text, default="[]")
    source = Column(String(200))
    source_url = Column(String(500))
    deadline = Column(DateTime)
    published_at = Column(DateTime, default=datetime.utcnow)
    required_tags = Column(Text, default="[]")
    
    # 关系
    plans = relationship("UserPlan", back_populates="opportunity")


class UserFeedback(Base):
    __tablename__ = "user_feedbacks"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    content_id = Column(Integer)
    content_type = Column(String(50))
    feedback_type = Column(String(50))
    created_at = Column(DateTime, default=datetime.utcnow)
    
    user = relationship("User", back_populates="feedbacks")


class ChatMessage(Base):
    __tablename__ = "chat_messages"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    role = Column(String(20))
    content = Column(Text)
    card_type = Column(String(50), nullable=True)
    card_data = Column(Text, nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    
    user = relationship("User", back_populates="chat_messages")


class UserPlan(Base):
    """用户计划"""
    __tablename__ = "user_plans"
    
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))
    opportunity_id = Column(Integer, ForeignKey("opportunities.id"))
    
    # 状态
    status = Column(String(50), default="pending")  # "pending" / "in_progress" / "completed" / "abandoned"
    
    # 时间节点
    added_at = Column(DateTime, default=datetime.utcnow)
    deadline = Column(DateTime)
    completed_at = Column(DateTime, nullable=True)
    abandoned_at = Column(DateTime, nullable=True)
    
    # 提醒设置
    reminder_enabled = Column(Boolean, default=True)
    last_reminder_at = Column(DateTime, nullable=True)
    
    # 备注
    notes = Column(Text, nullable=True)
    
    # 关系
    user = relationship("User", back_populates="plans")
    opportunity = relationship("Opportunity", back_populates="plans")
    reminders = relationship("PlanReminder", back_populates="plan")


class PlanReminder(Base):
    """计划提醒"""
    __tablename__ = "plan_reminders"
    
    id = Column(Integer, primary_key=True)
    plan_id = Column(Integer, ForeignKey("user_plans.id"))
    
    reminder_type = Column(String(50))  # "action" / "deadline" / "follow_up" / "abandon_suggest"
    content = Column(Text)
    
    scheduled_at = Column(DateTime)
    sent_at = Column(DateTime, nullable=True)
    
    is_read = Column(Boolean, default=False)
    
    # 关系
    plan = relationship("UserPlan", back_populates="reminders")
```
