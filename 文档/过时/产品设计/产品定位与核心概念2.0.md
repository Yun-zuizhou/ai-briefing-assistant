## 一、产品定位

### 1.1 产品名称

**简报助手App** —— 每天一份简报，帮你收集世界，记录自己，看见成长

### 1.2 产品定义

一款以"每日简报"为核心载体的个人信息管理应用，通过智能提炼将分散的信息整合为高度凝练的简报，主动帮助用户从信息海洋中获取有价值的内容，呈现信息的变化脉络，记录思考轨迹，见证个人成长。

### 1.3 核心概念

```
时代（热点/趋势/事件）←→ 我（观点/行动/成长）
```

**Slogan**：*"每天一份简报，帮你收集世界，记录自己，看见成长"*

### 1.4 定位优势

| 维度 | 分析 |
|------|------|
| **情感共鸣** | 每个人都想知道"我在时代中处于什么位置"，这是深层心理需求 |
| **差异化** | 不是冷冰冰的信息聚合，而是"成长陪伴者"的角色定位 |
| **故事性** | 用户的成长轨迹与时代变迁交织，形成独特的人生叙事 |
| **粘性** | 用户会想"一年后、五年后，我的观点变了吗？" |

## 二、核心价值主张

### 2.1 解决的核心问题

| 问题类型 | 具体表现 | 解决方案 |
|---------|---------|---------|
| **信息过载** | 信息分散在多个平台，需频繁切换 | 简报推送、一站式聚合 |
| **知识流失** | 获取信息后缺乏有效记录机制 | AI辅助整理、成长叙事 |
| **行动断层** | 停留在"知道"层面，缺乏行动转化 | 待办整合、智能提醒 |
| **成长不可见** | 无法看到自己的成长轨迹 | 个人叙事、趋势洞察 |

### 2.2 五大核心价值

| 价值 | 描述 |
|------|------|
| **智能提炼** | AI主动总结、加工信息，生成高度凝练的简报，而非简单推送原文 |
| **趋势洞察** | 呈现信息的变化脉络，提供周报、月报、季报等多维度趋势分析 |
| **意图识别** | 通过自然语言交互降低用户配置门槛，用户可直接提出需求 |
| **信息整理** | 整合用户零碎的想法和观点，转写成逻辑性强的叙述 |
| **用户画像** | 从信息关注、行动力等角度分析生成用户画像 |

## 三、目标用户与痛点

### 3.1 目标用户画像

| 用户画像 | 痛点 | 需求 |
|---------|------|------|
| **学术研究者** | 文献太多，难以追踪最新研究 | 自动推送相关领域文献 |
| **求职者** | 兼职信息分散，错过截止日期 | 集中推送 + 智能提醒 |
| **行业关注者** | 行业资讯分散在各平台 | 一站式信息聚合 |
| **自我成长者** | 想记录想法但缺乏动力 | AI辅助整理 + 成长可视化 |

### 3.2 用户使用场景

**场景示例**：用户在聊天框输入"帮我看看兼职的资料呗"，APP帮用户配置搜索、筛选合适的兼职资料，并添加追踪关注，添加到用户的"计划"卡组中，罗列兼职的要求、招聘截止日期等信息，并智能地给出建议。每天提醒，"简历通过了吗？""开始写简历了吗？"

## 四、核心功能设计

### 4.1 功能架构

```
┌─────────────────────────────────────────────────────────────┐
│                    简报助手App                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  每日简报（核心）                     │   │
│  │  • 信息追踪  • 待办提醒  • 个人叙事                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                         ↓                                   │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │  简报互动   │  │  简报管理   │  │  简报设置   │         │
│  │             │  │             │  │             │         │
│  │ • 回复记录  │  │ • 历史查看  │  │ • 推送时间  │         │
│  │ • 收藏内容  │  │ • 导出简报  │  │ • 内容偏好  │         │
│  │ • 创建待办  │  │ • 分享简报  │  │ • 关注管理  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 核心功能：每日简报

#### 4.2.1 简报结构

| 简报模块 | 内容 | 数据来源 |
|---------|------|---------|
| 📰 **信息追踪** | 热点、机会、学习资源 | 用户关注主题 + AI收集 |
| 📋 **待办提醒** | 今日待办、即将到期 | 用户创建的待办事项 |
| 📖 **个人叙事** | 今日记录、成长轨迹、AI洞察 | 用户行为数据 + AI生成 |

#### 4.2.2 简报推送机制

| 推送类型 | 时间 | 内容 |
|---------|------|------|
| **晨间简报** | 8:00 | 昨夜收集 + 今日待办 + 昨日叙事 |
| **晚间简报** | 21:00 | 今日收集 + 明日待办 + 今日叙事 |
| **即时简报** | 重要事件发生时 | 紧急提醒、重要机会 |

### 4.3 对话式交互

**所有用户输入都通过对话完成**，无需手动配置：

| 传统方式 | 简报助手App |
|---------|-----------|
| 手动填写表单设置关注主题 | "我想关注AI和远程工作机会" |
| 手动创建待办事项 | "明天提醒我投简历" |
| 手动记录想法 | "今天看到GPT-5的新闻，很有意思" |
| 手动设置推送时间 | "每天早上8点给我发简报" |

### 4.4 功能映射

| 功能模块 | "时代与我"视角下的定义 |
|----------|----------------------|
| 热点推送 | **时代脉搏** - 你正在经历的时代大事 |
| 信息收集 | **时代机遇** - 时代给你的机会窗口 |
| 观点记录 | **时代回响** - 你对时代的回应与思考 |
| 用户画像 | **时代坐标** - 你在时代中的位置与成长轨迹 |
| 计划管理 | **时代计划** - 追踪你的行动，陪伴你的成长 |

## 五、创新点

| 创新点 | 描述 |

## 七、技术架构

### 7.1 技术架构图

```
┌─────────────────────────────────────────────────────────────┐
│                        前端层                                │
│  React + TypeScript + Tailwind CSS + Vite                  │
├─────────────────────────────────────────────────────────────┤
│                        后端层                                │
│  Python + FastAPI + SQLAlchemy + Celery                    │
├─────────────────────────────────────────────────────────────┤
│                        AI层                                  │
│  DeepSeek API + LangChain                                   │
├─────────────────────────────────────────────────────────────┤
│                        数据层                                │
│  SQLite (MVP) / PostgreSQL                                  │
├─────────────────────────────────────────────────────────────┤
│                        基础设施                              │
│  Docker + CI/CD                                            │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 技术选型

| 层级       | 技术                       | 选型理由             |
| -------- | ------------------------ | ---------------- |
| **前端**   | React 18 + TypeScript    | 生态成熟，组件化开发，类型安全  |
| **前端样式** | Tailwind CSS             | 原子化CSS，快速构建响应式界面 |
| **前端构建** | Vite                     | 快速的开发服务器和构建工具    |
| **后端**   | Python + FastAPI         | AI生态丰富，高性能异步框架   |
| **ORM**  | SQLAlchemy               | 强大的ORM框架，支持异步操作  |
| **任务队列** | Celery                   | 分布式任务队列，用于定时推送   |
| **AI**   | DeepSeek API + LangChain | 能力强大，链式调用、记忆管理   |
| **数据库**  | SQLite / PostgreSQL      | 关系型数据库，支持JSON字段  |
| **开发环境** | Trae IDE                 | AI辅助设计，自然语言交互    |
| **打包**   | Capacitor (可选)           | 打包成原生App         |

### 7.3 MVP技术方案

| 模块 | MVP方案 | 完整方案 |
|------|---------|---------|
| 前端 | React + Tailwind | 同左 |
| 后端 | Python + FastAPI | 微服务架构 |

## 九、设计原则总结

### 9.1 产品设计原则

| 原则 | 说明 |
|------|------|
| **简报驱动** | 所有功能围绕简报展开 |
| **被动触达** | 推送机制，用户无需主动 |
| **互动闭环** | 回复简报即可完成操作 |
| **即时反馈** | 每个行为都有反馈 |
| **渐进引导** | 从简单到复杂 |

### 9.2 交互设计原则

| 原则 | 说明 |
|------|------|
| **对话式交互** | 所有输入通过对话完成 |
| **零学习成本** | 像聊天一样自然 |
| **即时响应** | AI理解意图并立即执行 |
| **灵活调整** | 随时对话修改配置 |

### 9.3 用户画像原则

| 原则 | 说明 |
|------|------|
| **动态演化** | 持续追踪用户行为，生成成长轨迹 |
| **第二人称伪装** | "你是一位..."、"你正在..." |
| **客观记录者语气** | 避免主观评价 |
| **成长导向** | 强调"正在成长"、"逐渐清晰" |

### 9.4 数据原则

| 原则 | 说明 |
|------|------|
| **观点数据** | 独立数据模型（UserOpinion） |
| **行为数据** | 反馈 + 观点 + 问答 |
| **综合评估** | 兴趣 + 行为 + 观点 |
