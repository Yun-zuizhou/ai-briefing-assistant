## 核心概念

**时代（热点/趋势/事件）←→ 我（观点/行动/成长）**

*Slogan：每天一份简报，帮你收集世界，记录自己，看见成长*

### 1.2 简报互动场景

#### 场景3：回复简报记录想法
```
触发：用户阅读简报后有想法

用户行为：
1. 用户回复："今天看到GPT-5的新闻，觉得AI发展太快了"

     📝 想法记录

     内容：看到GPT-5的新闻，觉得AI发展太快了
     标签：#AI #思考

- 记录你对时代的回应

```

```
- 自动关联原始内容
- 智能提醒不遗漏
- 将时代机遇转化为行动
```

## 二、用户引导场景

### 2.1 新用户首次使用

```
触发条件：用户首次打开APP

引导流程（对话式交互）：

[AI] 你好！我是你的简报助手 👋

     我每天会给你推送一份简报，包含：
     
     📰 信息追踪 —— 你关心的领域动态
     📋 待办提醒 —— 你的计划和提醒
     📖 个人叙事 —— 你的成长轨迹
     
     首先，你想关注什么？

用户输入："我想关注AI和远程工作机会"

[AI] 明白了！你关注：
     • AI技术动态
     • 远程工作机会
     
     你希望什么时候收到简报？

用户输入："每天早上8点"

[AI] 太好了！你的第一份简报将在明天早上8点送达。
     
     期待与你开始这段成长之旅！🌱
```

**对话式配置优势**：
- 零学习成本，像聊天一样自然
- AI理解意图并立即执行
- 无需手动填写表单

### 2.2 用户询问"我该做什么"

#### 情况A：有待办事项
```
用户输入："我今天该做什么？"

[AI] 看了看你的待办，今天有这些事：

     ☐ 投递简历（截止：明天）← 建议优先处理
     ☐ 完成项目报告（截止：后天）
     
     要现在开始处理吗？

用户输入："好，先投简历"
→ AI打开简历投递指引
```

#### 情况B：有新简报未读
```
用户输入："有什么新消息吗？"

[AI] 你的今日简报还没看呢！

     里面有3条新内容和2个待办提醒
     
     要现在查看吗？

用户输入："看看"
→ AI展示今日简报
```

#### 情况C：近期不活跃
```
用户输入："最近有什么？"

[AI] 有3天没见了！

     这几天有5条简报等你查看
     包括你关注的AI领域重要更新
     
     要看看吗？

用户输入："好的"
→ AI展示近期简报摘要
```

**对话式交互特点**：
- 用户无需学习界面操作
- 自然语言即可完成所有操作
- AI理解上下文，提供精准响应

## 八、场景设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **简报驱动** | 所有功能围绕简报展开 | 信息追踪、待办提醒、个人叙事统一在简报中 |
| **被动触达** | 推送机制，用户无需主动 | 每日定时推送简报 |
| **互动闭环** | 回复简报即可完成操作 | 回复"明天我要..."自动创建待办 |
| **即时反馈** | 每个行为都有反馈 | 回复简报→AI确认→更新叙事 |
| **渐进引导** | 从简单到复杂 | 先收简报→再互动→最后管理 |
| **对话式交互** | 所有输入通过对话完成 | "我想关注AI"→AI自动配置 |

## 九、功能模块与"时代与我"映射

> 注：以下映射展示了功能模块在"时代与我"概念框架下的意义，简报模块名称仍为：信息追踪、待办提醒、个人叙事

| 功能模块 | "时代与我"视角 | 使用场景 |
|----------|---------------|---------|
| 热点推送 | **时代脉搏** - 你正在经历的时代大事 | 晨间简报、即时简报 |
