# Railway 部署指南

## 1. 准备工作

### 1.1 注册 Railway 账号
- 访问 https://railway.app
- 使用 GitHub 账号登录

### 1.2 安装 Railway CLI（可选）
```bash
npm install -g @railway/cli
railway login
```

## 2. 部署后端服务

### 2.1 创建新项目
1. 在 Railway Dashboard 中点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择你的仓库
4. Railway 会自动检测 Python 项目

### 2.2 配置环境变量

在 Railway 项目的 Variables 标签中添加以下环境变量：

**必需变量**：
```
D1_DATABASE_ID=503a4b2f-b062-4ec5-be21-edfd0e7d8a2d
D1_ACCOUNT_ID=你的账户ID
D1_API_TOKEN=你的API Token
D1_USE_CLOUD_AS_SOURCE=true
SECRET_KEY=生产环境密钥（随机生成）
DEBUG=false
```

**AI 配置（选择一个）**：
```
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的DeepSeek API Key
```

**CORS 配置**：
```
CORS_ORIGINS=["https://你的前端域名.vercel.app","https://你的前端域名.pages.dev"]
```

### 2.3 生成 SECRET_KEY
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

### 2.4 部署
- Railway 会自动构建和部署
- 部署成功后会获得一个 URL，如：`https://xxx.up.railway.app`

## 3. 部署前端服务

### 3.1 选择前端托管平台
推荐选项：
- **Vercel**：与 React/Vite 集成最好
- **Cloudflare Pages**：与 D1 深度集成
- **Netlify**：简单易用

### 3.2 Vercel 部署步骤
1. 访问 https://vercel.com
2. 使用 GitHub 登录
3. 导入项目
4. 配置：
   - Root Directory: `prototype`
   - Build Command: `npm run build`
   - Output Directory: `dist`
5. 添加环境变量：
   ```
   VITE_API_ORIGIN=https://你的后端URL.up.railway.app
   ```

### 3.3 Cloudflare Pages 部署步骤
1. 访问 Cloudflare Dashboard
2. 进入 Pages
3. 连接 GitHub 仓库
4. 配置：
   - Root Directory: `prototype`
   - Build Command: `npm run build`
   - Build output directory: `dist`
5. 添加环境变量：
   ```
   VITE_API_ORIGIN=https://你的后端URL.up.railway.app
   ```

## 4. 配置 CORS

在后端 Railway 项目中更新 `CORS_ORIGINS`：
```json
["https://你的前端域名"]
```

## 5. 验证部署

### 5.1 后端验证
访问：`https://你的后端URL.up.railway.app/docs`
- 应该看到 FastAPI 文档页面

### 5.2 前端验证
访问：`https://你的前端域名`
- 应该看到应用首页

### 5.3 API 连接验证
在前端应用中测试 API 调用，确保数据正常加载

## 6. 常见问题

### 6.1 构建失败
- 检查 `requirements.txt` 是否完整
- 检查 Python 版本是否兼容

### 6.2 启动失败
- 检查环境变量是否正确配置
- 查看 Railway 日志排查错误

### 6.3 CORS 错误
- 确保 `CORS_ORIGINS` 包含前端域名
- 确保域名格式正确（包含 https://）

### 6.4 数据库连接失败
- 检查 D1 相关环境变量是否正确
- 确保 D1_USE_CLOUD_AS_SOURCE=true
