# Railway 部署步骤详解

## 第一步：准备 GitHub 仓库

1. 确保代码已推送到 GitHub
2. 确保 `.env.production` 已添加到 `.gitignore`，不会被提交

## 第二步：创建 Railway 项目

### 方式一：通过 Dashboard（推荐）

1. 访问 https://railway.app
2. 使用 GitHub 账号登录
3. 点击 "New Project"
4. 选择 "Deploy from GitHub repo"
5. 选择你的仓库
6. Railway 会自动检测 Python 项目

### 方式二：通过 CLI

```bash
# 安装 Railway CLI
npm install -g @railway/cli

# 登录
railway login

# 初始化项目
railway init

# 部署
railway up
```

## 第三步：配置环境变量

在 Railway 项目的 **Variables** 标签中添加以下环境变量：

### 必需变量

```bash
# D1 数据库配置
D1_DATABASE_ID=503a4b2f-b062-4ec5-be21-edfd0e7d8a2d
D1_DATABASE_NAME=ai-briefing-assistant-prod
D1_ACCOUNT_ID=你的账户ID
D1_API_TOKEN=你的API Token
D1_USE_CLOUD_AS_SOURCE=true

# 安全配置
SECRET_KEY=生成的随机密钥
DEBUG=false

# CORS 配置（JSON 数组格式）
CORS_ORIGINS=["https://你的前端域名"]
```

### AI 配置（选择一个）

```bash
# DeepSeek
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=你的DeepSeek API Key

# 或 OpenAI
AI_PROVIDER=openai
OPENAI_API_KEY=你的OpenAI API Key

# 或其他提供商...
```

### 生成 SECRET_KEY

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

## 第四步：部署

1. Railway 会自动构建和部署
2. 查看 **Deployments** 标签了解部署状态
3. 部署成功后会获得一个 URL，如：`https://xxx.up.railway.app`

## 第五步：验证部署

1. 访问 `https://你的URL.up.railway.app/docs`
2. 应该看到 FastAPI 文档页面
3. 测试几个 API 端点确保正常工作

## 第六步：配置自定义域名（可选）

1. 在 Railway 项目中进入 **Settings**
2. 点击 **Domains**
3. 添加自定义域名
4. 配置 DNS 记录

## 常见问题

### 构建失败

检查 `requirements.txt` 是否完整，确保所有依赖都已列出。

### 启动失败

查看 Railway 日志，检查环境变量是否正确配置。

### CORS 错误

确保 `CORS_ORIGINS` 包含前端域名，格式为 JSON 数组。

### 数据库连接失败

检查 D1 相关环境变量是否正确，确保 `D1_USE_CLOUD_AS_SOURCE=true`。
