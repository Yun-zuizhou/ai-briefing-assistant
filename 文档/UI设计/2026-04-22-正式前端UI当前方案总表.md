## 2. 当前唯一口径

### 2.1 产品定位

当前正式前端不是：

- 新闻站
- 通用 SaaS 后台
- 聊天机器人壳层
- 复古报纸展示稿

当前正式前端应统一理解为：

> 一个围绕“今日信息 -> 行动转化 -> 自我记录 -> 周期回顾”运行的移动端个人情报助手应用。

### 2.2 视觉方向

当前正式方向固定为：

- 温暖编辑感
- 纸张 / 档案感
- 克制而非花哨
- 信息秩序强于装饰冲击

明确停止继续扩散的方向：

- 蓝白科技风 / 通用 SaaS 风
- 全站统一 serif 正文
- 过度拟物化报纸主题
- 固定手机壳展示思路

### 2.3 当前页面结构主线

当前正式前端页面能力围绕以下主链路展开：

1. `Today`：今日信息入口
2. `Actions`：行动转化与跟进
3. `Chat`：最低成本表达入口
4. `Journal / Growth / Reports`：记录与周期回顾
5. `HotTopics / Article / Collections`：内容阅读与沉淀

## 3. 当前设计原则

### 3.1 内容优先

- 装饰只服务信息结构，不和正文争抢主次
- 一屏只能有一个主任务
- 主动作必须清楚，不能每块都像 CTA

### 3.2 标题与正文分治

- 标题、报头、栏目名：保留 serif/editorial 气质
- 正文、按钮、表单、标签、元信息：使用更理性的 sans 体系

当前代码虽未完全落地，但后续实现必须按这个方向收口。

### 3.2.1 移动优先

- 当前正式设计目标是 `WDB` 移动应用
- 桌面浏览器只保留开发调试与演示预览角色
- 不再把桌面端体验反向当成正式页面设计目标

### 3.3 正式页和研发态分离

以下内容不应继续出现在正式主链路页面中：

- 测试账号快捷入口
- 调试目标说明
- 会话缓存 / 候选意图 / 匹配来源等内部术语
- 面向研发的解释性段落

### 3.4 同类元素只允许一种正式写法

后续正式前端必须收口到统一组件系统，优先对象包括：

- `Button`
- `Card`
- `Input / Textarea`
- `Tag / Status`
- `SectionHeader`
- `Empty / Loading / Error State`

## 4. 当前执行顺序

后续 UI 迭代统一按下面顺序推进：

### 阶段 A：基础层

- 去掉固定手机壳展示
- 建立移动优先正式外壳
- 桌面浏览器仅保留居中预览兼容，不做桌面专用布局
- 收口 color / spacing / type / state tokens
- 建立标题 serif + 正文 sans 的双字体体系

### 阶段 B：组件层

- 让正式页面真正复用 `Button / Card / Input / Tag`
- 补齐 hover / focus / active / disabled / loading / selected 状态
- 停止新增页面内联样式块

### 阶段 C：样板页

按以下顺序打样：

1. `Welcome`
2. `Login`
3. `Today`
4. `Chat`

### 阶段 D：其余正式页面迁移

按以下顺序推进：

1. `HotTopics`
2. `Article`
3. `Collections`
4. `Actions`
5. `Growth / My`
6. `Reports`
7. `Settings`

### 阶段 E：体验与架构收尾

- 骨架屏
- 可访问性
- 响应式细节
- 错误边界
- 页面职责拆分

## 5. 前后端边界口径

### 5.1 纯前端可改

以下 UI 改动默认不需要改接口：

- 响应式壳层
- 排版、留白、配色、字体系
- 按钮 / 卡片 / 输入框 / 标签统一
- 页面分组、折叠、信息层级
- 空状态 / 加载态 / 错误态
- 研发态文案清理

### 5.2 需要同步契约或接口

以下场景默认必须同步检查 `packages/contracts` 与 `apps/edge-worker`：

- 页面需要展示后端当前未提供的字段
- 新增排序 / 筛选 / 搜索 / 分页
- 新增跨页面深链或动作语义
- 修改共享字段名、枚举值、状态值
- 重做 Chat 意图 / 确认 / 纠偏模型

## 7. 当前最容易再次混乱的点

### 7.1 旧品牌风格混入

失效口径包括：

- 蓝白主色
- Caveat / Quicksand
- 深蓝科技风 / 通用 APP 风

### 7.2 旧导航结构混入

失效口径包括：

- “首页 / 时代热点 / 成长机遇 / 我的画像”四 Tab 方案
- “对话式首页”作为总入口的早期结构表述

### 7.3 原型展示思路混入

失效口径包括：

- 固定手机壳
- 大量展示性装饰
- 页面为展示概念服务，而不是为主任务服务
