## 5. 图标资产状态

`apps/web/src/components/icons` 下有 26 个本地 SVG：

`arrow-left`、`bell`、`book-open`、`bookmark`、`chevron-down`、`chevron-right`、`chevron-up`、`clock`、`external-link`、`help-circle`、`home`、`info`、`log-out`、`message-square`、`pen-line`、`plus`、`search`、`send`、`settings`、`shield`、`sparkles`、`target`、`trash-2`、`trending-up`、`user`、`x`。

当前主线没有发现 `.svg` 或 `components/icons` 的 import。实际运行中的图标主要来自 `lucide-react`，例如：

| 使用点 | 来源 |
| --- | --- |
| `TabBar` 底部导航图标 | `lucide-react` 的 `Newspaper`、`TrendingUp`、`CheckSquare`、`FileText`、`MessageCircle`。 |
| `SecondaryHeader` 返回图标 | `lucide-react` 的 `ArrowLeft`。 |
| `Header` 内部图标 | `lucide-react` 的 `ArrowLeft`、`Share2`、`Settings`、`Bell`，但 `Header` 本身当前未接入页面。 |

结论：本地 SVG 图标是一组“已有但未接入”的资产。如果继续使用 `lucide-react`，可以归档或删除这批 SVG；如果希望固定视觉资产，应建立统一 Icon 组件并替换现有 lucide 调用。

## 6. 布局如何管理

主布局入口是 `PageLayout`：

| `variant` | 默认行为 | 当前页面类型 |
| --- | --- | --- |
| `main` | 显示底部 `TabBar` | `TodayPage`、`ActionsPage`、`GrowthPage`、`JournalPage`、`MyPage`、默认 `ChatPage`。 |
| `secondary` | 不显示底部导航 | 详情页、设置页、历史页、收藏页、热点页、关于/帮助页、AI 设置页等。 |
| `auth` | 不显示底部导航 | `WelcomePage`、`LoginPage`、`InterestConfigPage`、`PreviewPage`。 |
| `report` | 不显示底部导航 | `WeeklyReportPage`、`MonthlyReportPage`、`AnnualReportPage`。 |

布局 CSS 由全局样式管理：

- `main.tsx` 引入 `styles/design-tokens.css` 和 `index.css`。
- `index.css` 统一 import `shell.css`、`navigation.css`、`surfaces.css`、各页面样式文件。
- `shell.css` 管理 `.app`、`.page-content`、`.page-footer`、`.page-action-bar`、`.masthead` 等布局骨架。
- `navigation.css` 管理 `.bottom-nav` 和底部导航按钮。
- `design-tokens.css` 管理字体、颜色、边框、阴影、安全区等基础 token。
- 页面自身多数通过 className 连接全局 CSS，而不是 CSS module 或组件 scoped style。

当前布局特点：

- 应用根节点 `#root` 被限制为移动端宽度：`width: min(100%, 430px)`。
- 页面外壳是固定高度移动端容器：`100dvh`。
- 内容区通过 `PageContent` / `.page-content` 使用 `flex: 1` 和 `overflow: auto`。
- 主导航不是路由层统一挂载，而是通过 `PageLayout variant="main"` 决定是否显示。

## 7. 页面如何管理

- 使用 `BrowserRouter`、`Routes`、`Route`。
- 所有正式页面通过 `React.lazy` 懒加载。
- `Suspense` 使用 `RouteFallback` 作为路由加载态。

## 9. 整理建议

1. 先决定图标策略：保留 `lucide-react` 还是启用本地 SVG。两套同时存在会增加资产维护成本。
2. 将 `RouteFallback` 改为复用 `PageLoading`，或者删除未使用的 `LoadingSpinner` 资产。
3. 将页面内手写空状态逐步收口到 `EmptyState`，否则可以移除该组件。
4. 如果 `Card` / `Input` 是设计系统资产，应优先接入表单、列表、设置页；如果不是，应归档。
5. 评估 `Header`、`PageContainer`、`PageActionBar` 是否仍符合当前 `Masthead + PageContent` 布局体系，不符合则归档。
6. 补齐或删除 `pages/index.ts`。如果继续保留 barrel，应导出 `ChatPage`、`AiProviderSettingsPage` 并明确它是否是页面公共入口。
7. 将 demo 资产标记为“原型资产”，避免和主线组件混用；`formal-ui-copy` 建议归档或拆出对主线仍有价值的组件后删除副本。
