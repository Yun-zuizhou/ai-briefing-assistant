# Viewport Shell 标准

## 这轮先解决的问题

当前正式 UI 副本中存在典型的“样机锁死”问题：

- `html, body, #root` 被固定为 `932px`
- `#root` 被固定为 `max-width: 430px` 和 `height: 932px`
- `.app` 继续继承固定高度和隐藏溢出

这会导致：

1. 页面更像固定手机样机，而不是正式移动端应用
2. 在不同设备尺寸下缺乏响应式弹性
3. 桌面端预览和移动端正式承载混在一起

## 当前正式标准

### 1. 移动端优先

- 手机环境下，应用应自然铺满当前可用视口
- 不再写死为 `430 x 932`

### 2. 桌面端只做预览兼容

- 桌面环境可以居中显示
- 可以保留边框与阴影，帮助观察
- 但不能把桌面预览壳层反向当成正式移动端标准

### 3. 根节点规则

- `#root` 默认 `width: 100%`
- `#root` 默认 `min-height: 100dvh`
- 不允许继续固定 `height: 932px`

### 4. 应用壳层规则

- `.app` 必须形成视口级纵向壳层
- `.app` 默认使用 `height: 100dvh`
- 页面正文统一由 `main` 承担滚动
- 底部导航属于壳层固定区域，不应跟随正文滚出视口

### 5. 主滚动区规则

- `main` 使用 `flex: 1`
- `main` 必须可滚动
- `main` 需要 `min-height: 0`，避免被内容撑破 flex 壳层
- 底部导航不参与正文滚动

## 是否进入正式标准

是。

`Viewport Shell / App Shell` 不是局部页面技巧，而是所有正式页面共同依赖的基础结构标准。
